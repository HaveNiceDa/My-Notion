# 2026-05-30 CLI 登录态复用与失效提示优化

## 背景

用户希望 My-Notion CLI 首次使用时引导用户到 Web 设置弹窗生成默认 Token，并将该 Token 保存在本地登录态文件中。后续若用户没有特殊诉求，CLI 应默认复用本地登录态 Token，不需要来回复制；只有 Token 过期、撤销或失效时再提示用户重新获取。

## 实际核查

- CLI 已有本地配置文件：`~/.my-notion/config.json`。
- `auth login` 会将 `apiUrl` 和 `token` 写入本地配置，文件权限为 `0600`。
- 命令解析优先级为：命令行参数 > 环境变量 > 本地配置 > 默认线上 API URL。
- 需要优化的是首次缺 Token 与失效 Token 的用户提示，以及 Agent Skill 文档对本地登录态复用的说明。

## 本次改动

- `packages/my-notion-cli/src/config/store.ts`
  - 缺少 Token 时提示用户打开 My-Notion Web 设置页的 `API Token` 区域，复制默认 CLI Token。
  - 明确提示执行 `my-notion auth login --token <mnt_token>` 后会保存到 `~/.my-notion/config.json`，后续命令自动复用。
- `packages/my-notion-cli/src/client/http-client.ts`
  - 对 `TOKEN_EXPIRED`、`TOKEN_REVOKED`、`UNAUTHORIZED` 返回更明确的恢复建议。
  - 提示用户回 Web 设置页重置或复制最新默认 Token 后重新登录。
- `packages/my-notion-cli/src/index.ts`
  - CLI help 增加首次登录和本地配置文件说明。
- `packages/my-notion-cli/docs/usage.md`
  - 更新首次登录、本地登录态保存、何时需要重新复制 Token 的说明。
- `packages/my-notion-cli/README.md`
  - 补充默认 Token 会保存到本地并被后续命令复用。
- `packages/my-notion-skills/my-notion-shared/SKILL.md`
  - 更新 Agent 使用规则：不要每次都要求用户复制 Token，优先复用本地登录态。
  - Token 缺失、过期、撤销或无效时再引导用户重新复制并登录。
