# Phase 5 P0：机器 API 审计日志表

## 目标

为 `/cli/v1/*` Machine API 增加审计日志表，记录 requestId、tokenId、userId、endpoint、status 等关键字段，为后续问题追踪、限流和安全治理打基础。

## 已完成

- 新增 `cliAuditLogs` Convex 表：
  - `requestId`
  - `method`
  - `path`
  - `status`
  - `errorCode`
  - `requiredScope`
  - `tokenId`
  - `tokenPrefix`
  - `userId`
  - `durationMs`
  - `createdAt`
- 表索引：
  - `by_request_id`
  - `by_token`
  - `by_user`
  - `by_created_at`
- 新增内部 mutation `recordCliAuditLog`：
  - 仅内部调用，不暴露为公共 API。
  - 不记录 PAT 明文。
  - 不记录 token hash。
- 更新 `/cli/v1/*` HTTP action：
  - 成功响应写入审计日志。
  - 结构化错误响应写入审计日志。
  - 认证缺失、认证失败、权限不足、业务 404、内部 500 均会尝试落审计。
  - 审计写入失败只记录服务端错误，不影响原业务响应。
- 更新认证返回：
  - 对 revoked、expired、insufficient scope 等可解析 token 的失败场景，返回 `tokenId`、`userId`、`tokenPrefix` 供审计使用。
- 更新文档：
  - 架构文档 Phase 5 P0 标记审计日志完成。
  - CLI commands reference 说明审计字段和敏感信息边界。
  - 已运行 `pnpm sync:skills` 同步 `.trae/skills`。

## 安全边界

- 审计日志不记录 PAT 明文。
- 审计日志不记录 token hash。
- 审计日志 path 不包含 query string，避免记录搜索词等自由文本。
- 审计写入使用内部 mutation，不向外部 Agent 暴露查询能力。
