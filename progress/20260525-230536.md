# My-Notion MCP STDIO MVP

## 目标

进入 CLI/Skills 架构方案的 Phase 4，新增 `my-notion mcp serve --transport stdio`，让 MCP 客户端可以通过 My-Notion CLI 复用现有 PAT、HTTP client 和文档 CRUD 能力。

## 已完成

- 为 `@notion/my-notion-cli` 新增 MCP SDK 依赖：
  - `@modelcontextprotocol/sdk`
  - `zod` 升级到兼容 MCP SDK peer 范围的版本。
- 新增 `packages/my-notion-cli/src/mcp/server.ts`：
  - 使用 `McpServer` 与 `StdioServerTransport` 启动 STDIO MCP server。
  - 复用现有 `MyNotionClient`、`resolveApiUrl` 和 `resolveToken`。
  - 不通过 tool 参数传递或输出 PAT 明文。
- 新增 `packages/my-notion-cli/src/commands/mcp.ts`：
  - 支持 `my-notion mcp serve --transport stdio`。
  - 首版仅允许 `stdio` transport。
- 新增 4 个 MCP tools：
  - `my_notion_docs_search`：搜索当前 PAT 用户文档。
  - `my_notion_docs_fetch`：读取文档并返回 Markdown。
  - `my_notion_docs_create`：创建 Markdown 文档，`dryRun` 默认 `true`。
  - `my_notion_docs_update`：追加或覆盖更新，`dryRun` 默认 `true`。
- 每个 tool 返回：
  - `structuredContent`：机器可解析 JSON。
  - `content`：文本 fallback，格式化 JSON。
- 新增 `packages/my-notion-skills/my-notion-mcp/SKILL.md`：
  - 记录启动方式、tool schema、输出约定与安全规则。
- 更新 CLI 命令参考：
  - 增加 MCP 启动命令和 tool 列表。
  - 修正 pnpm dev 参数转发示例为 `pnpm --filter @notion/my-notion-cli dev <command>`。
- 运行 `pnpm sync:skills`，同步新增 `my-notion-mcp` 到 `.trae/skills/my-notion-mcp`。

## 验证结果

- `pnpm --filter @notion/my-notion-cli typecheck`：通过。
- `pnpm --filter @notion/my-notion-cli build`：通过。
- `pnpm --filter @notion/my-notion-skills lint`：通过，占位 lint 正常运行。
- `pnpm sync:skills`：通过，并同步：
  - `.trae/skills/my-notion-docs`
  - `.trae/skills/my-notion-mcp`
  - `.trae/skills/my-notion-shared`
- MCP STDIO smoke test：通过。
  - 使用 fake `MY_NOTION_API_URL` 和 `MY_NOTION_API_TOKEN` 初始化 server。
  - `tools/list` 正常返回 4 个 tools。
  - `tools/call my_notion_docs_create` 的 `dryRun: true` 正常返回 `structuredContent`，未触发真实网络写入。
- `GetDiagnostics`：无诊断错误。

## 已知风险

- 当前 MCP server 只支持 STDIO，不支持 HTTP transport、OAuth discovery 或远程 MCP 部署。
- `my_notion_docs_search` 的无 query 行为复用现有 search endpoint，是否等价于 list recent documents 取决于后端实现。
- `dryRun: false` 会直接调用真实 CLI API，MCP client 侧仍需提供用户确认 UX。
- 尚未加入 MCP 专用 E2E 脚本；当前仅通过 STDIO JSON-RPC smoke test 覆盖初始化、tool listing 和 dry-run tool call。
- CLI E2E 仍只清理测试 PAT，不清理测试文档。

## 下一步建议

- 增加 `scripts/e2e-my-notion-mcp.mjs`，用 JSON-RPC STDIO 覆盖 `initialize -> tools/list -> dryRun create/update -> search/fetch`。
- 增加文档清理能力，避免 CLI/MCP E2E 长期累积测试文档。
- 为 `/cli/v1/*` HTTP Actions 增加限流和审计日志。
- 若需要接入远程 MCP client，再单独设计 HTTP MCP + OAuth 2.1，不要和当前 STDIO MVP 混在一起。
