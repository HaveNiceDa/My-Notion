# Web Agent 文档写入 dry-run 闭环

## 本轮目标

- 开启 P0：`document_write` / `document_update` Web Agent dry-run。
- 复用 CLI/MCP 写文档安全契约：默认只预览，用户确认后才真实写入。
- 修复非法 `documentId` 直接触发 Convex validator 报错的问题，改为业务层可恢复错误。

## 已完成变更

- 新增 Web Agent 文档写入 tool：
  - `document_write`：创建新文档，参数为 `title`、`contentMarkdown`、可选 `parentDocument`。
  - `document_update`：更新当前文档或指定文档，支持 `append` / `overwrite` 和可选标题更新。
  - 两个 tool 均只返回 `dryRun: true`、`confirmationRequired: true` 和 Markdown 预览，不在 LLM tool 执行阶段写库。
- 新增前端确认落库流程：
  - `ToolCallCard` 支持文档创建/更新预览。
  - 用户点击“应用变更”后调用 Convex mutation 写入。
  - 用户取消或应用后，会把 tool result 状态写回 AI 消息。
- 新增 Convex 文档 Markdown 写入 mutation：
  - `createFromMarkdown`
  - `updateFromMarkdown`
  - `documentId` / `parentDocument` 使用字符串入参，在 handler 内通过 `ctx.db.normalizeId` 校验，避免非法字符串触发 Convex 参数 validator 报错。
- 抽取 Markdown / BlockNote 转换 helper：
  - CLI/MCP 与 Web 写入 mutation 复用同一套 `markdownToBlockNoteJson`、`appendMarkdownToBlockNoteJson`、`blockNoteJsonToMarkdown`。
- 更新 i18n：
  - 补充中英文文档写入/更新 tool、确认、取消、失败、打开文档等文案。
- 更新计划文档：
  - `docs/ai-chat-refactor-plan.md` 已标记 Web Agent 文档写入 dry-run + 前端确认落库完成。

## 已知风险

- Markdown 到 BlockNote 的转换仍是基础段落级转换，尚未保留标题、列表、表格等复杂 Markdown 结构。
- 文档写入确认 UI 已具备最小闭环，但尚未做 Playwright E2E 覆盖。
- 新建文档后不会自动跳转或刷新侧边栏树，需要用户点击“打开文档”进入新文档。

## 下一步建议

1. 做 P1 Agent 性能监控：记录 ReAct 轮次、tool latency、LLM latency 和失败降级 trace。
2. 补 `web_extract` 和 `document_search` 两个高收益只读 tool。
3. 为文档写入确认流补最小 E2E 或组件级测试。
