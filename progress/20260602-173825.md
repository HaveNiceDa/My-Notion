# 2026-06-02 17:38:25 M19 Plan 模式最小闭环

## 目标

- 基于已有 `task_plan` tool，补齐“生成计划 -> 用户确认 -> 执行步骤 -> 状态可见”的最小产品闭环。
- 按当前决策移除 Spec 模式作为后续主线，避免路线继续分叉。

## 完成变更

- 在 AI Chat 输入框增加 Plan 模式切换，Plan 模式请求会携带 `mode: "plan"`。
- `/api/agent/stream` 在 Plan 模式下只暴露 `task_plan` tool，并通过 system prompt 明确禁止确认前执行写入或持久化操作。
- `task_plan` 卡片增加“确认执行”入口，用户确认后会把结构化计划转成执行 prompt，以普通 Chat 模式继续交给 Agent 执行。
- 执行按钮具备本地状态反馈：待确认、启动中、已开始执行，并提示后续执行过程会在新回复中展示。
- 更新中英文 i18n 文案，避免 Plan 模式新增 UI 文案硬编码。
- 更新路线文档，移除 Spec 模式作为后续主线，M20 收敛为 Web Agent MCP adapter。

## 已知缺口

- 当前执行状态是前端本地状态，刷新后不会恢复为“已开始执行”；后续如果需要跨会话恢复，可扩展 `aiMessages.toolResults` 的 plan 状态回写。
- 步骤级执行进度暂由 Agent 新回复自然呈现，尚未做每个 step 的细粒度状态事件。
