# Tool 展示位置与 Agent 日志降噪

## 本轮目标

- 调整 AI Chat tool 卡片位置：只读信息源放在回答正文前，需要用户确认的写类 tool 保留在正文后。
- 降低服务端 Agent 日志噪音，避免 ReAct / Trace 明细默认刷屏。
- 明确 `document_search` 的搜索范围和返回结构。

## 已完成变更

- `MessageList`：
  - 将 tool 结果分为两类展示。
  - `knowledge_search`、`web_search`、`web_extract`、`document_search`、`document_read`、`memory_read` 放在 AI 正文上方。
  - `document_write`、`document_update`、`memory_write` 放在 AI 正文下方，保证确认按钮靠近最终说明。
  - 保留重复只读 tool 折叠逻辑。
- `AgentTracer`：
  - `[AgentTrace]` 结构化日志默认不输出。
  - 需要排查时设置 `AGENT_TRACE_LOG=1` 或 `AGENT_TRACE_LOG=true` 打开。
- `runReActLoop`：
  - ReAct 明细日志默认不输出。
  - 需要排查时设置 `AGENT_DEBUG_LOG=1` 或 `AGENT_DEBUG_LOG=true` 打开。
  - 未知 tool 和达到最大工具轮次的 warning 仍保留。
- `context-compression`：
  - 上下文压缩开始/完成日志也接入 `AGENT_DEBUG_LOG`。
  - 摘要失败的 error 日志仍保留。

## document_search 说明

- 能搜索：
  - 当前登录用户自己的 `documents`。
  - 文档标题 `title`。
  - 层级路径 `path`，例如 `项目 / AI Chat / M17`。
  - 最近编辑时间，支持 `updatedAfter`。
  - 空 `query` 时返回最近编辑文档。
- 默认不搜索：
  - 文档正文内容。
  - 向量语义内容。
  - 其他用户文档。
  - 归档文档，除非 `includeArchived: true`。
- 返回字段：
  - `documentId`、`title`、`path`、`parentDocument`。
  - `isArchived`、`isStarred`、`isInKnowledgeBase`。
  - `updatedAt`、`createdAt`。
  - `metadata.count`、`metadata.searchedCount`、`metadata.includeArchived`、`metadata.updatedAfter`。
