## 已知缺口

- `task_plan` 目前是基础 tool 和展示能力，还不是完整 Plan 模式；尚未实现用户确认计划后逐步执行。
- 业务内 validation error 仍有部分旧工具只返回 `{ error, recoverable }`，后续可继续补齐 `summary/sources/metadata`。
- Harness 相关 Storybook、Memory eval、real retrieval eval、Tool Trace Replay 按新优先级后置。

## 下一步建议

1. 做 Plan 模式最小闭环：`task_plan` 生成计划 -> 用户确认 -> 步骤状态展示/推进。
2. 做 Spec 模式最小闭环：复杂写入前先生成结构化规格，确认后再执行写类 tool。
3. 做 Web Agent MCP adapter：复用现有 CLI/MCP 能力扩展工具生态。
