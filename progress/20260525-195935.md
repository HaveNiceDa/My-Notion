# Web 设置页 PAT 管理 UI

## 目标

在 Web 设置弹窗中增加 Personal Access Token 管理入口，消费已有 `GET /api/cli/tokens` 与 `DELETE /api/cli/tokens` 接口，让用户可以查看和撤销 CLI / Agent 使用的 PAT。

## 已完成

- 扩展 `SettingsModal`：
  - 打开设置弹窗时自动加载 PAT 列表。
  - 支持手动刷新。
  - 展示 token 名称、前缀、状态、scope、创建时间、上次使用时间和过期时间。
  - 对已撤销 token 禁用撤销按钮。
  - 撤销前使用 `AlertDialog` 二次确认。
  - 撤销成功后局部更新列表中的 token 状态。
- 接入现有接口：
  - `GET /api/cli/tokens`
  - `DELETE /api/cli/tokens?tokenId=...`
- 增加错误处理：
  - 列表加载失败时显示 inline error 并触发 toast。
  - 撤销失败时触发 toast。
- 补充中英文 i18n：
  - `packages/business/i18n/en.json`
  - `packages/business/i18n/zh-CN.json`

## 验证结果

- `pnpm --filter @notion/web exec tsc --noEmit`：通过。
- `pnpm --filter @notion/web lint`：通过，仍有 7 个历史 warning，非本次新增。
- `pnpm --filter @notion/web build`：通过，Sentry auth token/source map 为现有 warning。

## 已知风险

- 当前 UI 只支持查看和撤销 PAT，暂不支持创建 PAT。
- 撤销接口依赖 Clerk 登录态，用户未登录或 Convex token 获取失败时会返回错误。
- 列表使用一次性加载，未做自动轮询或分页；当前 PAT 数量预期较小可接受。

## 下一步建议

- 在设置页增加 PAT 创建表单，消费现有 `POST /api/cli/tokens`，并仅在创建成功后展示一次明文 token。
- 增加复制 token 明文的安全提示和“我已保存”确认流程。
- 后续再进入 MCP Phase 4。
