# Web 设置页 PAT 创建表单

## 目标

在已有 PAT 管理 UI 中补齐创建能力，消费 `POST /api/cli/tokens`，并保证新创建的明文 token 只展示一次，避免用户误关闭导致无法再次查看。

## 已完成

- 扩展 `SettingsModal` 的 PAT 管理区域：
  - 新增 token 名称输入框。
  - 新增“创建 Token”按钮。
  - 创建时调用 `POST /api/cli/tokens`。
  - 创建成功后刷新 token 列表。
- 增加一次性明文 token 展示流程：
  - 新 token 明文只保存在组件临时 state。
  - 使用醒目的提示区展示明文 token。
  - 提供复制按钮，调用浏览器 Clipboard API。
  - 用户必须勾选“我已复制并安全保存”后，才能隐藏明文 token。
  - 明文 token 未确认保存前，阻止关闭设置弹窗并用 toast 提示。
- 保持安全边界：
  - token 列表仍只展示 `tokenPrefix`，不展示历史明文。
  - 创建成功后隐藏明文 token 会清空组件内的临时 token state。
- 补充中英文 i18n 文案：
  - `packages/business/i18n/en.json`
  - `packages/business/i18n/zh-CN.json`

## 验证结果

- `pnpm --filter @notion/web exec tsc --noEmit`：通过。
- `pnpm --filter @notion/web lint`：通过，仍有 7 个历史 warning，非本次新增。
- `pnpm --filter @notion/web build`：通过，Sentry auth token/source map 为现有 warning。

## 已知风险

- 复制依赖浏览器 Clipboard API；若浏览器或权限不支持，会显示复制失败提示，用户仍可手动选中复制。
- 创建表单当前只支持 token 名称，scope 和过期时间仍使用后端默认值。
- 尚未增加专门的 E2E/UI 自动化覆盖。

- 可补 Playwright 用例覆盖设置页 PAT 创建、复制提示和撤销流程。
