# AI Smoke 脚本收口

## 本次目标

- 新增 `ci:ai-smoke` 脚本，把 Web unit、Agent unit、retrieval eval 和 AI Chat mock E2E 串成一个入口。

## 已完成

- 新增 `scripts/ci-ai-smoke.mjs`：
  - `Web unit`：执行 AI Chat 组件/流客户端测试。
  - `Agent unit`：执行 `apps/web/src/lib/agent/__tests__` 下的 Agent 后端测试。
  - `Retrieval eval`：执行 `pnpm eval:retrieval`。
  - `AI Chat mock E2E`：执行认证态 Playwright mock E2E。
- 更新根 `package.json`：
  - 新增脚本 `"ci:ai-smoke": "node scripts/ci-ai-smoke.mjs"`。
- E2E 环境策略：
  - 默认缺少 `CLERK_SECRET_KEY` 时跳过 AI Chat mock E2E，避免本地 smoke 被认证前置阻塞。
  - 设置 `AI_SMOKE_REQUIRE_E2E=1` 时，缺少 `CLERK_SECRET_KEY` 会硬失败，适合 CI 严格校验。
  - 如果 `PLAYWRIGHT_BASE_URL` 不可达，脚本会自动启动 `@notion/web` dev server；设置 `AI_SMOKE_START_SERVER=0` 可关闭自动启动。
- 更新 `docs/ai-chat-refactor-plan.md`：
  - 将 `ci:ai-smoke` 标记为脚本完成。
  - 保留 AI smoke GitHub Actions、认证环境 E2E 验证、Memory eval 等后续缺口。
