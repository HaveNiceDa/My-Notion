# Phase 5 P0：机器 API 基础限流

## 目标

为 `/cli/v1/*` Machine API 增加首版基础限流，按 PAT 维度保护读写接口，避免 token 滥用或异常 Agent 循环造成服务压力。

## 已完成

- 新增 `cliRateLimits` Convex 表：
  - `tokenId`
  - `endpointKey`
  - `windowKey`
  - `count`
  - `windowStart`
  - `expiresAt`
  - `updatedAt`
- 表索引：
  - `by_token_endpoint_window`
  - `by_expires_at`
- 新增内部 mutation `checkAndIncrementCliRateLimit`：
  - 使用固定窗口计数。
  - 默认窗口为 60 秒。
  - 返回 `allowed`、`limit`、`remaining`、`resetAt`。
- 更新 `/cli/v1/*` HTTP action：
  - 认证通过后、业务处理前执行限流。
  - 限流维度为 `tokenId + method + endpoint`。
  - 动态文档路径归一化为 `/cli/v1/documents/:id`。
  - 超限时返回 HTTP `429` 和结构化错误码 `RATE_LIMITED`。
  - 限流错误继续带 `requestId`，并写入 `cliAuditLogs`。
  - 响应 header 返回 `Retry-After`、`x-ratelimit-limit`、`x-ratelimit-remaining`、`x-ratelimit-reset`。
- 更新 CLI client：
  - `RATE_LIMITED` 不再被本地重试，避免放大限流压力。
- 更新文档：
  - `packages/my-notion-skills/my-notion-docs/references/cli-commands.md` 增加限流规则说明。
  - `docs/my-notion-cli-skills-architecture.md` 和 `.trae/documents/my-notion-cli-skills-architecture-plan.md` 标记基础限流完成。
  - 已运行 `pnpm sync:skills` 同步 `.trae/skills`。

## 首版限流规则

- 读接口：每个 token 每分钟 120 次。
- 写接口：每个 token 每分钟 30 次。
- `GET /cli/v1/auth/status`：每个 token 每分钟 60 次。
- `POST /cli/v1/tokens/revoke-current`：每个 token 每分钟 10 次。
