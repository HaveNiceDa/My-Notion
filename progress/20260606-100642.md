# 流式续跑 Phase 1/2/3

## 目标

- 完成 `docs/agent-stream-resume-protocol.md` 中 Phase 1/2/3 的最小可运行闭环。
- 在已输出内容后支持基于 `runId/seq` 的 backlog replay。
- 对失败 run 提供保守 checkpoint 恢复，并复用已完成 tool result，避免重复执行副作用或写入预览工具。

## 已完成

- Phase 1：在 Agent stream 协议中加入 `run-start`、`checkpoint`、`resume-start`、`resume-unavailable` 控制事件。
- Phase 1：前端 `stream-client` 支持 `AgentStreamResumeCursor`，记录 `runId`、`lastAppliedSeq`、`assistantMessageId`，并在 resume 请求中从已有 cursor 继续推进。
- Phase 1：`useAIChatStream` 将 resume cursor 写入 `sessionStorage`，并在 assistant 消息落库后把临时消息 ID 替换为真实消息 ID。
- Phase 2：新增 Convex `agentRuns`、`agentRunEvents`、`agentRunCheckpoints` schema 和 `create/append/finish/backlog/latestCheckpoint` 函数。
- Phase 2：新增 `AgentRunRecorder`，统一分配流式 `seq`，向前端输出 NDJSON，并异步持久化 event/checkpoint。
- Phase 2：`/api/agent/stream` 支持 resume 请求，按 `seq > lastAppliedSeq` replay backlog；completed run 补发 `finish`。
- Phase 3：`runReActLoop` 支持 `resumeToolResults`，从 checkpoint 注入 run-local cache。
- Phase 3：失败 run 可读取最近 checkpoint，恢复压缩消息、模型、thinking 配置和已完成 tool result，继续 ReAct Loop。
- Phase 3：补充测试覆盖 resume cursor、`resume-unavailable` 和 checkpoint tool result 复用，确保写入类 tool 不被重复执行。

## 已知缺口

- running run 的 live stream 接管尚未实现；当前 resume 可 replay backlog，并对 failed/completed run 做处理。
- checkpoint 只保存 `currentDocument` 的 `id/title/contentHash` 摘要，恢复时尚未重新拉取完整当前文档上下文。
- 前端已持久化 resume cursor，但还未提供显式“继续生成”按钮入口。
- Phase 4 Trace Replay UI、trace sink 关联和可视化重放仍待后续推进。

## 下一步

- 补 UI 层“继续生成”入口，读取 `sessionStorage` 中最近 cursor 并发起 resume 请求。
- 在 resume 时按 `currentDocument.id` 安全拉取最新文档上下文，恢复 document_read/document_update 等依赖当前文档的工具。
- 将 `agentRunEvents` 与 trace sink 关联，进入 Phase 4 Trace Replay。
