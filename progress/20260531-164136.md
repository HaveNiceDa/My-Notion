# 2026-05-31 My-Notion CLI 浏览器授权登录

## 背景

- 旧 CLI 登录要求用户从 Web 设置页复制 `mnt_` 明文 Token，再交给 CLI 或 Agent。
- 这会让 Agent 接触长期凭据，不符合“用户不感知 Token，只点击链接授权一次”的目标。
- 参考飞书 CLI 的 Device Flow 交互后，本次将主登录链路调整为浏览器授权，并引入 profile 隔离线上与本地调试登录态。

## 本次改动

- CLI 配置从单 `{ apiUrl, token }` 升级为 `version: 2` 的 `profiles` 模型，并兼容旧配置自动迁移。
- `my-notion auth login` 默认启动 Device Flow，输出授权链接；`--token` 保留为 legacy/CI 后门。
- 新增 Device Flow 客户端逻辑，支持 `--profile`、`--local`、`--web-url`、`--no-open`、`--no-wait`、`--device-code`。
- 新增 Convex `cliDeviceAuthSessions` 表和创建、批准、拒绝、轮询、消费会话逻辑。
- 新增 Web API：`/api/cli/auth/device/start|approve|deny|poll`。
- 新增 Web 授权页：`/[locale]/cli/auth`，用户登录后可批准或拒绝 CLI 访问。
- Device Flow 创建的 token 只写 `tokenHash`，不写 `tokenPlaintext`，降低服务端明文长期存储风险。
- 更新 CLI README、usage 和 My-Notion Skills 认证说明。
