# CLI PAT 自助撤销与本地登出

## 目标

按 CLI/Skills 阶段计划补齐 PAT 生命周期的终端侧闭环：支持本地清理已保存 token，并允许当前 PAT 通过机器 API 自助撤销。

## 已完成

- 新增 `my-notion auth logout`：
  - 清除 `~/.my-notion/config.json` 中保存的 `token`。
  - 保留已保存的 `apiUrl`，方便后续重新登录。
  - 输出 `loggedOut`、`hasToken` 和 `configPath`，不打印 token 明文。
- 新增 `my-notion tokens revoke-current`：
  - 通过当前 Bearer PAT 调用机器 API。
  - 仅撤销当前认证 token，不接受客户端传入 `userId` 或任意 token id。
  - 返回 token 元信息，包括 `id`、`tokenPrefix`、`scopes`、`lastUsedAt`、`revokedAt`。
- 新增 Convex HTTP endpoint：
  - `POST /cli/v1/tokens/revoke-current`
  - 复用现有 PAT hash 校验流程。
  - 对该 endpoint 不要求 `docs:write` scope，避免“低权限 token 无法撤销自己”。
- 更新 `scripts/e2e-my-notion-cli.mjs`：
  - 端到端流程从 8 步扩展为 9 步。
  - 使用 CLI 自身执行 `tokens revoke-current`。
  - 撤销后验证 `auth status` 失败。
  - 最后执行 `auth logout` 验证本地 token 已清除。
- 更新 `my-notion-shared` 和 `cli-commands.md`：
  - 补充 `auth logout` 与 `tokens revoke-current` 的命令说明。
  - 同步 `.trae/skills/my-notion-*`。

## 已知风险

- `tokens revoke-current` 只能撤销当前 PAT；任意 token 的 list/revoke 仍依赖 Web 登录态 API 或后续 UI。
- E2E 仍只清理临时 PAT，不删除测试文档。
- Web 设置页的 PAT 管理 UI 还未实现。
