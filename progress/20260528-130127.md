# Web Extract 与 Document Search 工具

## 本轮目标

- 扩展 Web Agent 两个 P1 只读工具：
  - `web_extract`：给定 URL 抽取网页标题、描述和正文。
  - `document_search`：按标题、路径和最近编辑时间搜索用户文档元数据。

## 已完成变更

- 新增 `web_extract`：
  - 落点：`apps/web/src/lib/agent/tools/web-extract.ts`。
  - 支持 `http` / `https` URL 校验。
  - 拦截 localhost、私网 IP、`.local` 等本地网络地址，降低 SSRF 风险。
  - 使用 `fetch` + `AbortController`，默认 12 秒超时。
  - 清理 `script`、`style`、`noscript`、`svg` 和 HTML 标签，输出轻量正文。
  - 正文最多返回 12000 字符，并标记 `metadata.truncated`。
  - 支持向前端推送 `tool-result-delta` 预览。
- 新增 `document_search`：
  - Agent tool 落点：`apps/web/src/lib/agent/tools/document-search.ts`。
  - Convex 查询落点：`packages/convex/documents/logic/searchMetadata.ts`。
  - 支持按标题、层级路径搜索。
  - 支持空 query 返回最近编辑文档。
  - 支持 `limit`、`includeArchived`、`updatedAfter`。
  - 仅返回文档元数据和路径，不返回正文，避免与 `knowledge_search` 职责重叠。
- Tool registry 更新：
  - `web_extract` 和 `document_search` 已作为基础只读工具始终可用。
- 更新 `docs/ai-chat-refactor-plan.md`：
  - `web_extract` / `document_search` 标记完成。
  - M17 主线标记为完成。
  - 下一步优先级调整为 Memory 增强、RAG 质量补齐、Trace sink 增强。

## 已知风险

- `web_extract` 当前是基础 HTML 清洗，不具备浏览器渲染能力；依赖客户端渲染的页面可能抽不到完整正文。
- 私网拦截目前覆盖常见 hostname/IP 字面量，未做 DNS 解析后的私网 IP 校验。
- `document_search` 为避免重查询成本，最多读取用户最近 500 个未归档/可选归档文档后做内存过滤；超大文档库后续可增加专用索引或分页。

## 下一步建议

1. 手动验证 Agent 对“总结这个 URL”是否优先调用 `web_extract`。
2. 手动验证 Agent 对“帮我找最近的项目计划文档”是否调用 `document_search`。
3. 下一阶段建议做 Memory 增强或 RAG context packing / citation quality。
