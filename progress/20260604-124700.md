# Agent 记忆高级能力清理

## 目标

在上一轮“软废弃”基础上继续做减法，移除当前文档/画板业务用不到的复杂 Agent 记忆操作，以三类业务记忆为最新主链路。

## 完成内容

- 删除不再使用的复杂 UI 组件：
  - `MemoryOverview.tsx`
  - `MemoryConflicts.tsx`
  - `MemoryDetailDrawer.tsx`
- 移除 `memory_read` 兼容工具，只保留 `memory_search` 和 `memory_write`。
- 精简 `memory_search` 参数，删除 `kinds/categories/scopes/includeSensitive/includeEvidence` 等高级过滤入口。
- 精简 `memory_write` 参数，删除 `expiresAt/supersedesMemoryId` 等复杂治理入口。
- 精简 Tool 卡片展示，删除 duplicate/conflict 提示和 `scoreBreakdown` 等复杂返回字段。
- 精简前端记忆类型，只保留业务需要字段：类型、内容、来源、原因、证据原文、置信度、状态和时间。
- 精简 Convex 记忆操作参数：
  - `create/propose/update/commit` 不再接收 kind/category/scope/privacy/importance/stability/reviewDue/embeddingStatus/supersedes 等高级参数。
  - `propose` 不再执行重复/冲突检测。
  - `list/listPending` 不再返回复杂治理字段。
- 清理中英文 i18n 中的复杂记忆治理文案。
- 更新测试，使用最新 `memory_search` 主链路。

## 当前记忆链路

- 前端页面：待确认、已记住、规则设置。
- 记忆类型：用户偏好、项目规则、最近决策。
- Agent 工具：`memory_search` 读取已确认记忆，`memory_write` 提议写入待确认。
- 自动提取：仅显式信号，仍进入待确认。
- 后端写入：保留基础 derived defaults，但不再暴露复杂治理参数。

## 后续建议

- 如确认不需要历史复杂字段，可单独规划 Convex schema 迁移清理。
- 如果后续移动端接入记忆页面，应直接复用当前三段式精简模型。
