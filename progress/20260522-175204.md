# Agent Auto 模式与移除显式 RAG 开关

## 完成的改动

- 将 AI Chat 前端统一切到 Agent `auto` 模式，不再从输入框显式传递 `chat` / `rag` 模式。
- 移除 `MessageInput` 中的 RAG 知识库 icon 和 `onModeChange` 交互。
- 移除 `useAIChat` 内部的 `mode` 状态、`setMode` 方法和相关请求参数链路。
- 移除 `AIChatPanel` 向 `MessageInput` 透传的 `mode` / `onModeChange` props。
- 删除 `models.ts` 中已不再使用的 `CHAT_MODES` / `ChatMode` 类型。
- 后端 `/api/agent/stream` 支持 `auto` 模式：根据用户 query 信号自动判断是否执行 `knowledge_search`。
- 后端保留 `chat` / `rag` 请求值兼容，但默认缺省进入 `auto`。
- `knowledge_search` 失败时降级为 tool result，不再直接中断整段 Agent 响应。
- 清理 `zh-CN` / `en` 中已不再使用的 RAG 开关文案。

## 验证命令和结果

- `pnpm --filter @notion/web exec tsc --noEmit`: ✅
- `pnpm --filter @notion/web lint`: ✅ 通过，有 4 个既有 warning
- `pnpm --filter @notion/web build`: ✅ 通过，有 Sentry auth token 和 edge runtime 常规 warning
- `GetDiagnostics`: ✅ 新增改动无错误；`useAIChat` 仍保留既有 `convex` effect 依赖 warning

## 已知缺口或风险

- `auto` 当前使用启发式 query 信号判断是否检索知识库，尚不是完整 LLM tool planning。
- `document_read` tool 尚未接入，因此“总结/翻译/深度剖析当前页面”仍无法直接读取当前编辑器文档内容。
- 旧 `/api/rag-stream`、`/api/rag-complete`、`/api/chat` 仍保留，后续可在 Agent 主路径稳定后清理。

## 下一步建议

- 接入 `document_read` tool，把当前文档内容作为 Agent 可调用能力。
- 将 `auto` 策略从启发式升级为通用 tool planner，但继续避免 DashScope thinking mode 下使用 object/required `tool_choice`。
- 清理旧 RAG API 和旧 `ragUtils.ts` 中残留的 thinking step 逻辑。
