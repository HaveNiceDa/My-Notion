# M21 韧性与治理最小闭环

## 目标

进入 M21，补齐 Web Agent 的三项基础治理能力：流式重试、Tool 结果契约基建、Plan 状态增强。

## 已完成

- 流式客户端增加请求级重试：
  - 默认最多重试 1 次。
  - 仅在 fetch 失败或尚未收到任何流式事件时重试。
  - 已收到部分 token / tool 事件后中断不重试，避免重复输出。
  - 408/5xx 可重试，429 保留 `Retry-After` 友好错误。
- 新增 `tool-result-v1` 结果契约基建：
  - 新增 `result-contract.ts`。
  - fallback 错误、文档写入预览、task_plan、MCP adapter metadata 开始带 `contractVersion`。
  - 保持业务 result 结构不大改，避免破坏现有 Tool 卡片。
- Plan 状态增强：
  - `task_plan` 确认执行后将 `planExecutionStatus: "started"` 写回 `aiMessages.toolResults`。
  - `ToolCallCard` 可从已持久化状态恢复“已开始执行”状态。
  - Convex `updateToolResultState` 支持 `task_plan` 状态写回。
- 补充测试：
  - 流式网络失败重试。
  - 已收到部分事件后不重试。
  - Tool metadata 契约版本。
  - Plan 已执行状态恢复。

## 已知限制

- 流式重试仅覆盖“尚未收到事件”的安全场景；已输出内容后的恢复续跑仍未实现。
- Tool 结果契约先统一 metadata，不强制迁移全部 tool 的 result shape。
- Plan 状态只记录“已开始执行”，尚未做步骤级事件流、逐 step 持久化和跨会话恢复。

- 补 Trace Sink / Tool Trace Replay，为 M22 Harness 回补做准备。
