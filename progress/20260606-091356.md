# Tool 结果契约全量统一

## 目标

在 M21 基础上继续收敛 Web Agent Tool 结果契约，让所有主要 tool result 稳定携带 `summary`、`sources`、`metadata` 和 `recoverable`。

## 已完成

- 扩展 `result-contract.ts`：
  - 新增 `withToolResultContract`，用于在保留业务字段的前提下补齐统一契约字段。
  - 新增 `buildToolErrorResult`，统一可恢复错误结果。
  - 继续沿用 `tool-result-v1` 作为契约版本。
- 迁移主要 Web Agent tools：
  - `knowledge_search`
  - `document_search`
  - `document_read`
  - `document_write`
  - `document_update`
  - `web_search`
  - `web_extract`
  - `memory_search`
  - `memory_write`
  - `task_plan`
  - `mcp_my_notion_call`
- 保持前端兼容：
  - Tool 业务字段如 `documents`、`results`、`memory`、`document`、`markdown` 保持不变。
  - 新字段作为 envelope 补充，不破坏 `ToolCallCard` 现有读取路径。
- 补充测试：
  - `tools.test.ts` 覆盖主要工具的契约字段。
  - `document-read.test.ts` 覆盖 `document_read` 成功与错误契约。

## 已知限制

- 当前契约统一以 Web Agent tools 为主，`packages/ai` 旧 tool registry 尚未迁移。
- `sources` 目前为轻量引用结构，后续可进一步定义强类型 discriminated union。
- 历史已保存消息中的旧 tool result 不会自动迁移，前端仍保持兼容读取。
