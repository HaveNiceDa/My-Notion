# Agent 记忆机制精简

## 目标

将 My-Notion 当前偏通用 Agent 的复杂记忆治理页面和工具描述，收敛为更适合文档/画板业务的基础记忆机制。

## 完成内容

- 将 `/memories` 主页面精简为三段结构：待确认、已记住、规则设置。
- 将产品层记忆类型收敛为三类业务记忆：用户偏好、项目规则、最近决策。
- 隐藏主 UI 中的复杂治理字段，包括 kind/category/scope、embeddingStatus、privacy、usageCount、importance、reviewDue 等。
- 精简记忆表单，移除置信度输入，内部继续默认传 `confidence: 1` 以保持后端兼容。
- 保留 Inbox 安全链路：Agent 提议的记忆仍需用户确认后才会生效。
- 收敛 `memory_search` / `memory_write` 工具描述，使模型围绕偏好、项目规则和最近决策使用记忆。
- 保守保留自动提取，只从明确用户信号中提取，并补充文档/画板规则分类。
- 更新 system prompt 中的记忆注入文案，弱化通用 Memory System 心智。
- 更新中英文 i18n 文案和相关测试。

## 已知取舍

- 本次不删除 Convex schema 字段，避免迁移风险。
- `MemoryOverview`、`MemoryConflicts`、`MemoryDetailDrawer` 暂不从文件系统删除，作为软废弃组件保留，主页面不再引用。
- `memory_search` 的高级参数仍保留兼容，但工具描述不再鼓励模型主动使用复杂过滤。

## 后续建议

- 如后续确认复杂治理组件长期不用，可单独清理软废弃组件和旧 i18n key。
- 若移动端后续接入记忆页面，应直接复用三类业务记忆模型，不引入通用 Agent Memory Center。
