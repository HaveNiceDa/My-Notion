# Memory Review UI

## 本轮目标

- 实现长期记忆集中管理界面。
- 支持查看、搜索、按类型过滤、编辑和停用已保存的 Agent Memory。

## 已完成变更

- 新增 Convex mutation：
  - `updateAgentMemory` 支持编辑 `type`、`content`、`reason`、`confidence`、`expiresAt`。
  - 仅允许编辑当前用户自己的 `active` 记忆。
  - 空内容会拒绝保存，置信度会裁剪到 `0-1`。
- 调整 `listAgentMemories`：
  - 最大返回数量从 20 提升到 100，满足 Review UI 列表展示。
  - Agent tool 层仍保留自己的最大 20 条限制。
- 新增 Memory Review UI：
  - 路由：`/[locale]/memories`。
  - 页面组件：`MemoryReviewPage`。
  - 支持搜索、类型过滤、编辑、保存、取消编辑和停用。
  - 停用复用 `deactivateAgentMemory` 软删除能力。
- 更新左侧导航：
  - 新增“长期记忆”入口。
- 更新中英文 i18n：
  - 新增 `MemoryReview` 文案。
  - 新增 `Navigation.memories`。
- 运行 `convex codegen` 更新 Web Convex generated types。

## 已知缺口

- Review UI 暂未提供“新建手动记忆”，目前主要管理 Agent 已保存的长期记忆。
- 停用操作没有二次确认弹窗，点击后会直接软删除。
- 保存/取消状态仍未回写到历史 tool result，刷新聊天历史后 memory write 确认卡片可能再次显示。

## 下一步建议

1. 给停用操作增加确认弹窗，避免误删。
2. 支持在 Review UI 中手动新增记忆。
3. 将 Memory 自动注入 Agent loop，减少 LLM 忘记调用 `memory_read` 的概率。
