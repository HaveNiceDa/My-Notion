# web_search tool 接入 + M14 bug 修复

## 完成的改动

### web_search tool 接入

- 新增 `lib/agent/tools/web-search.ts`：`executeWebSearch` 通过 DashScope `enable_search` 联网搜索
- 更新 `lib/agent/tools/definitions.ts`：新增 `webSearchTool` 定义（含 query + strategy 参数）
- 更新 `lib/agent/tools/registry.ts`：`web_search` 始终可用
- 更新 `lib/agent/tools/index.ts`：导出 `webSearchTool` 和 `executeWebSearch`

### web_search 实现策略

- LLM 自主决定调用 `web_search` function tool
- tool 内部发起一次带 `enable_search: true` 的 LLM 调用（使用 `qwen3.6-plus`）
- 搜索结果作为 tool result 返回给 ReAct 循环
- 支持三种搜索策略：`turbo`（默认）、`max`、`agent`

### M14 bug 修复

- 修复 `document_read` tool：移除 LLM 无法知道的 `documentId` 必填参数，改为无参数 tool
- 修复前端消息构建：恢复当前用户消息的添加（之前误删），修复历史消息 JSON 格式问题
- 添加 ReAct 循环日志：`[ReAct]` 前缀，便于调试

## 验证命令和结果

- `pnpm --filter @notion/web exec tsc --noEmit`: ✅
- `pnpm --filter @notion/web build`: ✅
- `pnpm --filter @notion/web lint`: ✅（4 个既有 warning）

## 已知缺口或风险

- web_search 内部使用 `qwen3.6-plus` 固定模型，后续可考虑与用户选择的模型对齐
- DashScope 联网搜索有额外计费（agent/agent_max 策略每次调用额外收费）
- 旧 RAG API 路由（`/api/rag-stream`、`/api/rag-complete`、`/api/chat`）仍存在
