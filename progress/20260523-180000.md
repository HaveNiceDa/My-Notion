# 清理旧 RAG API 路由 + web_search tool 接入 + M14 bug 修复

## 完成的改动

### 清理旧 RAG API 路由

- 删除 `/api/chat/route.ts` — 旧聊天路由
- 删除 `/api/rag-stream/route.ts` — 旧 RAG 流式路由
- 删除 `/api/rag-complete/route.ts` — 旧 RAG 完成路由
- 删除 `/api/embeddings/route.ts` — 旧 embedding 路由（Agent tool 直接使用 @notion/ai/server）
- 删除 `/api/qdrant/route.ts` — 旧 Qdrant 路由（Agent tool 直接使用 @notion/ai/server）
- 删除 `lib/rag/ragUtils.ts` — 旧 RAG 工具库（含硬编码 thinking step 逻辑）
- 精简 `rag-documents/route.ts` — 移除 `runRAGQuery` action 和对 ragUtils 的依赖

### API 路由变化

之前 15 个路由 → 现在 5 个路由：
- `/api/agent/stream` — Agent 统一入口（ReAct 循环）
- `/api/rag-documents` — 知识库文档管理
- `/api/editor-ai/streamText` — 编辑器 AI
- `/api/edgestore/[...edgestore]` — 文件存储
- `/api/upload-image` — 图片上传

### web_search tool 接入

- 新增 `lib/agent/tools/web-search.ts`：通过 DashScope `enable_search` 联网搜索
- 更新 definitions/registry/index：注册 `webSearchTool`
- `ToolContext` 新增 `model` 字段，web_search 使用用户选择的模型

### M14 bug 修复

- 修复 `document_read` tool：移除 LLM 无法知道的 `documentId` 必填参数
- 修复前端消息构建：恢复当前用户消息的添加，修复历史消息 JSON 格式问题
- 添加 ReAct 循环日志

## 验证命令和结果

- `pnpm --filter @notion/web exec tsc --noEmit`: ✅
- `pnpm --filter @notion/web build`: ✅
- `pnpm --filter @notion/web lint`: ✅（4 个既有 warning）

## 已知缺口或风险

- web_search 内部使用非流式 LLM 调用，可能增加延迟
- DashScope 联网搜索有额外计费
