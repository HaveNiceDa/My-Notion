# Phase 5 P0：CLI 基础重试与超时策略

## 目标

缓解 CLI/MCP E2E 中偶发的网络层 `fetch failed`，为 My-Notion CLI Machine API client 增加基础超时、指数退避和有限重试能力。

## 已完成

- 更新 `packages/my-notion-cli/src/client/http-client.ts`：
  - 每次请求最多尝试 3 次。
  - 单次请求 10 秒超时，超时后通过 `AbortController` 中断。
  - 重试使用指数退避，起始等待 300ms。
  - 对网络失败、超时、HTTP 408、HTTP 429 和 HTTP 5xx 进行重试。
  - 对结构化 4xx 业务错误不重试，例如 `UNAUTHORIZED`、`TOKEN_REVOKED`、`TOKEN_EXPIRED`、`INSUFFICIENT_SCOPE`、`NOT_FOUND`。
- 更新 `packages/convex/cli/index.ts`：
  - `archiveCliDocument` 改为幂等行为。
  - 对已归档且属于当前 PAT 用户的文档，重复 archive 会返回已归档文档，不再返回 404。
- 更新文档：
  - `packages/my-notion-skills/my-notion-docs/references/cli-commands.md` 增加 Retry and Timeout 章节。
  - `docs/my-notion-cli-skills-architecture.md` 和 `.trae/documents/my-notion-cli-skills-architecture-plan.md` 标记该 P0 待办完成。
  - 已运行 `pnpm sync:skills` 同步 `.trae/skills`。

## 当前结论

- CLI Machine API client 已具备基础网络韧性。
- `fetch failed` 仍可能在全部重试失败后出现，但短暂网络抖动会被自动重试吸收。
- 后续如仍频繁出现网络层失败，可继续增加可配置超时/重试参数，或在 E2E 脚本层增加整段流程重试。
