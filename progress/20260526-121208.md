# Phase 5 P1：CLI 文档导入导出

## 目标

实现 `my-notion docs export` 和 `my-notion docs import`，让 Agent 可以将 My-Notion 文档导出为本地 Markdown，并将本地 Markdown 文件导入为新文档。首版仅扩展 CLI，不新增 MCP tool。

## 已完成

- 新增 CLI 命令 `docs export`：
  - `my-notion docs export --id <documentId> --format markdown`
  - `my-notion docs export --id <documentId> --output <file.md> --format markdown`
  - `my-notion docs export --id <documentId> --output <file.json> --format json`
  - 未传 `--output` 时输出到 stdout。
  - 传入 `--output` 时写入本地文件，并在 stdout 返回包含 `id`、`title`、`output`、`format`、`bytes` 的摘要。
- 新增 CLI 命令 `docs import`：
  - `my-notion docs import --title <title> --file <file.md> --format json`
  - `--content-file` 可作为 `--file` 的别名。
  - 首版复用现有创建文档 API，导入会创建新文档，不修改已有文档。
- 更新 CLI help：
  - 增加 `docs export` 和 `docs import` 用法。
- 更新 CLI E2E：
  - 创建测试文档后导出 Markdown 文件。
  - 校验导出内容包含原始关键字和追加内容。
  - 从导出的 Markdown 再导入新文档。
  - 归档原始文档和导入文档。
  - 撤销临时 PAT 并清理本地登录状态。
- 更新 Skill 文档：
  - `my-notion-docs/SKILL.md` 增加 import/export 使用场景。
  - `references/cli-commands.md` 增加 import/export 精确命令参考。
  - 已运行 `pnpm sync:skills` 同步 `.trae/skills`。
- 更新架构文档：
  - `docs/my-notion-cli-skills-architecture.md` 和 `.trae/documents/my-notion-cli-skills-architecture-plan.md` 中 Phase 5 P1 导入导出任务已标记完成。
