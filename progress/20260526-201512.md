## 决策

- 当前不做 `my-notion kb search`。
- 当前不把 RAG/Qdrant 搜索能力接入 CLI。
- `kb search` 仅在后续产品目标需要“Agent 检索知识库再回答/生成”时启动。
- 若后续启动 RAG 检索，应先把 Web Agent 的 `knowledge_search` 逻辑下沉到 `packages/ai/server`，避免 CLI 直接依赖 `apps/web/src`。
