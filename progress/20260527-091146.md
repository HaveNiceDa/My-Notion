# Hybrid Retrieval Service MVP

## 本次目标

按 M17 的混合检索策略开始落地代码，实现 `knowledge_search` 的默认 `balanced` 混合搜索 MVP。

## 已完成

- 新增 `packages/ai/server/retrieval/` 模块，包含统一类型、`retrieveKnowledge(options)`、语义召回、关键词召回、metadata 召回和 RRF 融合排序。
- 在 `QdrantVectorStoreWrapper` 中新增 `keywordSearch()` 和 `metadataSearch()` 两个只读检索方法。
- 保留 `fast` 策略为单路 Qdrant 语义检索。
- 将 `balanced` 策略实现为 semantic / keyword / metadata 三路召回 + fusion/dedup。
- 将 `deep` 策略暂时复用 balanced 流程，并保留策略标识，后续接 query rewrite、multi-query、rerank 和 context packing。
- 改造 Web Agent 的 `knowledge_search` tool，默认调用 `retrieveKnowledge({ strategy: "balanced" })`。
- 扩展 `knowledge_search` tool schema，允许 LLM 显式选择 `fast`、`balanced`、`deep`。
- 新增 `packages/ai/__tests__/retrieval.test.ts`，覆盖融合去重、fast 策略、balanced 三路召回。
- 更新 `apps/web/src/lib/agent/__tests__/tools.test.ts`，覆盖默认 balanced 和显式 strategy 参数。

## 已知风险

- 第一版 keyword recall 使用 Qdrant payload scroll + 轻量 token match，适合作为 MVP；文档量变大后应替换为 BM25、全文索引或专用搜索服务。
- 第一版 metadata recall 主要依赖现有 Qdrant payload 中的 `title`、`documentId`、`chunkIndex`，对 `tags`、`updatedAt` 的效果取决于后续索引字段补齐。
- `fast` 暂未支持 `filters.documentIds` 的 include 语义，避免误用旧 `excludeDocumentIds` 参数；后续需要给 Qdrant semantic search 增加 include filter。
- `deep` 当前只是接口占位，尚未实现 query rewrite、multi-query search、rerank 和 context packing。

## 下一步建议

1. 给 Qdrant 语义检索补 include filter，避免 `filters.documentIds` 只能在 keyword/metadata 生效。
2. 索引时补充 `updatedAt`、`tags`、层级标题、chunk 邻近摘要等 metadata，提升 metadata recall 和 contextual chunk 效果。
3. 为 `deep` 策略增加 query rewrite 和 multi-query search。
4. 增加 retrieval trace 事件，为后续 M18 harness / eval 做回放准备。
