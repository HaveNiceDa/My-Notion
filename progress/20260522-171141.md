# 修复 Agent RAG Thinking 与 Tool Choice 兼容性

## 完成的改动

- 修复 RAG 模式开启深度思考时报错：`tool_choice parameter does not support being set to required or object in thinking mode`。
- 调整 `/api/agent/stream` 的 RAG 首轮调用：强制选择 `knowledge_search` tool 时不再传 `extra_body.enable_thinking`。
- 保持第二轮答案生成继续使用用户选择的 thinking mode，因此不影响最终回答的深度思考能力。
- 在代码中补充注释，记录 DashScope thinking mode 与 object 形式 `tool_choice` 的兼容性限制。

## 验证命令和结果

- `pnpm --filter @notion/web exec tsc --noEmit`: ✅
- `pnpm --filter @notion/web build`: ✅ 通过，有 Sentry auth token 和 edge runtime 常规 warning
- `GetDiagnostics`: ✅ `/api/agent/stream/route.ts` 无诊断错误

## 已知缺口或风险

- 首轮 tool 选择不启用 thinking，仅用于生成 tool call；第二轮生成最终答案时才启用 thinking。
- 如果后续改成多 tool Agent loop，需要继续遵守 DashScope thinking mode 下 `tool_choice` 的限制。

- 在 RAG + thinking 场景下手动复测一次侧边栏对话。
- 后续接入 `document_read` tool 时复用该兼容策略。
