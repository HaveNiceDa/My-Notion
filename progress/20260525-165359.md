# My-Notion Skills 首版文档

## 目标

为 `packages/my-notion-skills` 编写首版 Skill 文档，让 Agent 能理解如何配置并调用 My-Notion CLI 来管理文档。

## 已完成

- 新增 `my-notion-shared/SKILL.md`：
  - 说明 CLI 运行方式、PAT/API URL 配置、输出格式和安全规则。
  - 明确 Agent 默认使用 `--format json`，读取正文时使用 `--format markdown`。
  - 记录 `pnpm e2e:cli` 作为 CLI 端到端验证入口。
- 新增 `my-notion-docs/SKILL.md`：
  - 说明文档创建、读取、搜索、列表和更新工作流。
  - 约定创建/更新复杂内容时优先使用临时 Markdown 文件。
  - 约定追加和覆盖的安全边界。
- 新增 `my-notion-docs/references/cli-commands.md`：
  - 覆盖 `auth login/status`。
  - 覆盖 `docs create/fetch/search/list/update`。
  - 给出 Agent 推荐调用模式和 E2E 验证说明。

## 验证结果

- `pnpm --filter @notion/my-notion-skills lint`：通过，占位 lint 正常运行。
- `find packages/my-notion-skills -maxdepth 3 -type f | sort`：确认文件结构正确。
- `Grep` 检查 `SKILL.md` frontmatter：确认两个 Skill 均包含 `name` 和 `description`。
- `GetDiagnostics`：无诊断错误。

## 已知风险

- 这些 Skill 当前位于 `packages/my-notion-skills`，尚未接入自动安装/同步到 Agent 的 `.trae/skills` 目录。
- `my-notion-mcp` 仍为空骨架，待后续 MCP 能力设计后补充。
- 当前 CLI 尚未实现终端侧 token list/revoke 命令，Skill 中仅记录了现状。

## 下一步建议

- 增加一个 skills 打包/同步脚本，将 `packages/my-notion-skills/*/SKILL.md` 同步到可被 Agent 发现的位置。
- 为 CLI 增加 `tokens list/revoke` 命令，完善终端侧 PAT 管理闭环。
