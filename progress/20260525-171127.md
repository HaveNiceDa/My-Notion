# My-Notion Skills 同步脚本

## 目标

增加同步脚本，将 `packages/my-notion-skills/*/SKILL.md` 及其引用文件同步到 Agent 可发现的 `.trae/skills/<skill-name>` 目录。

## 已完成

- 新增 `scripts/sync-my-notion-skills.mjs`：
  - 扫描 `packages/my-notion-skills` 下包含 `SKILL.md` 的子目录。
  - 校验 `SKILL.md` frontmatter 中的 `name` 和 `description`。
  - 校验目录名与 frontmatter `name` 一致。
  - 同步完整 skill 目录到 `.trae/skills/<skill-name>`。
  - 仅覆盖本次同步的目标 skill，不影响其他 `.trae/skills` 下已有 skill。
- 根目录 `package.json` 新增 `sync:skills` 脚本。
- `packages/my-notion-skills/package.json` 新增 `sync` 脚本。
- 已同步以下 skill：
  - `.trae/skills/my-notion-docs`
  - `.trae/skills/my-notion-shared`

## 验证结果

- `node --check scripts/sync-my-notion-skills.mjs`：通过。
- `pnpm sync:skills`：通过。
- `pnpm --filter @notion/my-notion-skills sync`：通过。
- `find .trae/skills -maxdepth 3 -type f | sort`：确认同步结果存在。
- `Grep` 检查 `.trae/skills/**/SKILL.md` frontmatter：通过。
- `pnpm --filter @notion/my-notion-skills lint`：通过，占位 lint 正常运行。
- `GetDiagnostics`：无诊断错误。

## 已知风险

- 同步脚本会整体覆盖目标 skill 目录，因此不要在 `.trae/skills/my-notion-*` 下手动修改未回写到 `packages/my-notion-skills` 的内容。
- 当前只同步包含 `SKILL.md` 的目录，因此 `my-notion-mcp` 空骨架不会被同步。

## 下一步建议

- 为 CLI 增加 `tokens list/revoke` 命令，完善终端侧 PAT 管理。
- 后续可将 `sync:skills` 纳入开发检查或文档发布流程。
