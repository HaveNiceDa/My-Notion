# My-Notion CLI 端到端验证脚本

## 目标

生成测试 PAT Token，并构造可复用脚本完整验证 `auth login -> docs create -> docs fetch -> docs update -> docs search` 流程。

## 已完成

- 新增 `scripts/e2e-my-notion-cli.mjs`。
- 新增根命令 `pnpm e2e:cli`。
- 脚本会自动：
  - 构建 `@notion/my-notion-cli`。
  - 生成 `mnt_` 前缀测试 PAT。
  - 使用 Convex CLI `--identity` 将 token hash 写入 dev 部署的 `apiTokens` 表。
  - 使用临时 `HOME` 跑 `my-notion auth login`，避免污染本机默认 CLI 配置。
  - 创建测试 Markdown 文档。
  - 读取文档并校验 Markdown 内容。
  - 追加更新文档并校验内容。
  - 搜索唯一关键词并校验能命中文档。

## 已知风险

- 测试 PAT 和测试文档目前会留在 dev Convex 部署中，后续需要补 token revoke/delete 和测试清理能力。
- 脚本会在最终 JSON 中输出完整测试 token，便于人工复用；该 token 仅应用于 dev 环境，避免用于生产。
