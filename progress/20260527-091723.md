# Hybrid Retrieval 注释与语义过滤补齐

## 本次目标

给混合检索执行链路补充关键注释，并进入下一步：为语义检索补齐 `documentIds` include filter 支持。

## 已完成

- 在 `retrieveKnowledge()` 执行链路补充注释，说明 `fast`、`balanced`、`deep` 的执行边界。
- 在 `retrieveBalanced()` 补充注释，说明三路召回并发执行与 RRF 融合的原因。
- 在 `fuseCandidates()` 补充注释，说明为何使用 RRF 处理异构召回分数。
- 在 `QdrantVectorStoreWrapper` 中新增 `semanticSearch()`，支持 `includeDocumentIds` 与 `excludeDocumentIds`。
- 保留旧 `similaritySearch()` 的兼容行为，继续把旧的第四个参数视为 exclude filter。
- 将 `semanticRecall()` 改为调用 `semanticSearch()`，并把 `filters.documentIds` 作为 include filter 传入。
- 更新 `packages/ai/__tests__/retrieval.test.ts`，覆盖三路召回都收到 `documentIds` 过滤参数。

## 验证

- `pnpm --filter @notion/ai typecheck`: 通过。
- `pnpm vitest run packages/ai/__tests__/retrieval.test.ts`: 通过，6 个用例。
- `pnpm --filter @notion/web test src/lib/agent/__tests__/tools.test.ts`: 通过，25 个用例。
- `pnpm --filter @notion/web exec tsc --noEmit`: 通过。
- `pnpm --filter @notion/web build`: 通过。
- `GetDiagnostics`: 无诊断问题。

## 已知风险

- `hybridSearch()` 名称仍保留旧命名以减少破坏性改动，但当前更准确的语义入口是 `semanticSearch()`。
- Qdrant filter 的 include/exclude 组合目前只覆盖 documentId，后续如果接 tags、workspace、updatedAt，需要继续扩展 filter builder。

## 下一步建议

1. 索引时补充 `updatedAt`、`tags`、层级标题、chunk 邻近摘要等 metadata。
2. 将 `metadataSearch()` 从当前轻量标题/ID/tag 匹配升级为更完整的结构化召回。
3. 开始实现 `deep` 策略的 query rewrite 和 multi-query search。
