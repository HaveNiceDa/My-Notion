# Agent Stream 与知识库 Tool 最小闭环

## 完成的改动

- 新增 `/api/agent/stream`，作为 AI Chat 的统一 Agent 流式入口。
- 定义 Agent NDJSON 事件协议：`text-delta`、`reasoning-delta`、`tool-call-start`、`tool-call-delta`、`tool-call-result`、`finish`、`error`。
- 在 Agent API 内实现 `knowledge_search` tool，RAG 模式会先强制调用知识库检索，再把 tool 结果交给模型生成最终回答。
- `knowledge_search` 复用 `@notion/ai/server` 的 `getOrCreateVectorStore` 和 Qdrant `similaritySearch`，不再走旧的硬编码 thinking step pipeline。
- 将 Web AI Chat 前端从 `/api/rag-stream` 切换到 `/api/agent/stream`。
- `useAIChat` 增加 Agent 事件解析和 tool call 状态更新。
- `MessageList` 增加轻量 tool 调用状态展示，不恢复旧的硬编码思考步骤。
- 补充 `zh-CN` / `en` 中 tool 状态展示文案。

## 已知缺口或风险

- 旧 `/api/rag-stream`、`/api/rag-complete`、`/api/chat` 仍保留，当前只完成前端主路径切换，尚未删除旧 API。
- `knowledge_search` 当前依赖已有 Qdrant collection 内容；如果知识库未初始化或未同步文档，检索结果会为空。
- Agent API 当前只内置 `knowledge_search`，`document_read` 和 `web_search` 尚未纳入新路由。
- Tool 调用状态为轻量展示，尚未做可展开参数/结果详情。
- RAG 模式采用强制首轮 tool call，后续可改为更通用的多 tool Agent loop。

## 下一步建议

- 接入 `document_read` tool，让“总结/翻译/深度剖析此页面”读取当前文档内容。
- 将知识库文档同步/初始化沉淀到 `knowledge_search` tool 内，避免空 collection 时体验不稳定。
- 新 Agent route 稳定后，删除或重定向旧 `/api/rag-stream`、`/api/rag-complete`。
- 将 tool 调用记录保存到 Convex 或与 assistant message 关联，支持历史对话回放。
