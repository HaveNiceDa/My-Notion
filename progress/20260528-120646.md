# AI Chat 写入确认与滚动体验修复

## 本轮目标

- 修复文档写入成功后，确认状态写回 `aiChat:updateToolResultState` 时因临时 `messageId` 触发 Convex validator 报错。
- 调整深度思考与消息流式输出的自动滚动策略。
- 将 tool 调用卡片默认改为展开状态。

## 已完成变更

- 修复 `messageId` 临时 ID 问题：
  - `saveMessage` 现在返回真实 `aiMessages` ID。
  - assistant 消息持久化成功后，本地消息 ID 会从 `Date.now()` 临时 ID 替换为真实 Convex ID。
  - `MessageList` 在渲染 tool 确认按钮时会识别纯数字临时 ID；临时 ID 不再传给 `updateToolResultState`，避免 `v.id("aiMessages")` 校验报错。
- 优化自动滚动：
  - 消息列表默认保持贴底，流式内容和深度思考增量会自动滚到底部。
  - 用户手动向上滚动后，暂停自动滚动，优先尊重用户阅读位置。
  - 用户滚回底部附近后，自动滚动恢复。
- 优化 tool 展示：
  - `ToolCallCard` 默认展开，文档写入预览和确认按钮默认可见。
- 清理滚动职责：
  - 自动滚动逻辑从 `AIChatPanel` 下沉到 `MessageList` 的滚动容器内，避免父组件无条件抢滚动。

- 自动滚动的“接近底部”阈值当前为 40px，后续可按实际体验微调。
