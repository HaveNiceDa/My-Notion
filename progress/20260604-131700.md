# Agent 记忆旧数据迁移清理

## 目标

清理 Convex `agentMemories` 表中遗留的复杂记忆字段，使数据库数据与精简后的 schema 保持一致，解决 `Schema validation failed`。

## 执行过程

- 临时恢复旧字段为 optional，使 Convex 能加载旧文档并执行 mutation。
- 新增一次性 mutation `cleanupLegacyAgentMemoryFields`。
- Dry-run 扫描：

```bash
pnpm --filter @notion/web exec convex run agentMemories:cleanupLegacyAgentMemoryFields '{"dryRun":true,"limit":5000}'
```

结果：

```json
{
  "cleaned": 1,
  "dryRun": true,
  "hasMore": false,
  "normalizedStatus": 0,
  "scanned": 1
}
```

- 执行真实清理：

```bash
pnpm --filter @notion/web exec convex run agentMemories:cleanupLegacyAgentMemoryFields '{"dryRun":false,"limit":5000}'
```

结果：

```json
{
  "cleaned": 1,
  "dryRun": false,
  "hasMore": false,
  "normalizedStatus": 0,
  "scanned": 1
}
```

- 清理完成后删除一次性 mutation，并将 `agentMemories` schema 恢复为最终精简版。
- 执行 `convex ai-files update` 更新 Convex AI 文件。

## 验证

```bash
pnpm --filter @notion/web exec convex dev --once
```

结果：通过，`Convex functions ready`，不再出现 `Schema validation failed`，也不再提示 AI files 过期。

```bash
pnpm --filter @notion/web typecheck
pnpm --filter @notion/ai typecheck
pnpm --filter @notion/web test -- ai-chat-components memory-extractor tools tool-result-cache
```

结果：全部通过，11 个测试文件、112 个测试通过。
