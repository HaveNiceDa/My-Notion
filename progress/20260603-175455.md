# 2026-06-03 M24 Agent Memory Retrieval Runtime

## 本次目标

执行 `milestones/M24-agent-memory-retrieval-runtime.md`，在 M23 schema 基础上重构 Agent Memory 读取与运行时注入机制，移除读路径 upsert，引入 `memory_search`，并将自动注入收敛为 compact instruction memory。

## 完成内容

- 更新 `packages/ai/server/memory.ts`：
  - `retrieveRelevantMemories` 不再在读路径调用 `upsertAgentMemory`。
  - 保留 Qdrant `semanticSearchAgentMemories` 纯读检索。
  - Qdrant 不可用或无命中时回退到 `fallbackRankMemories`。
  - semantic 与 fallback 结果都输出 `matchScore` 和 `scoreBreakdown`。
  - 轻量排序引入 semantic、scope、importance、confidence、recency、usage、staleness penalty。
- 更新 `packages/ai/rag/qdrantVectorStore.ts`：
  - Agent Memory 向量索引内容和 metadata 支持 M23 新字段：`kind/category/scope/summary/importance`。
  - 写路径同步仍保留，读路径不再负责补索引。
- 更新 `apps/web/src/lib/agent/tools/memory.ts`：
  - 新增 `executeMemorySearch`。
  - 支持 `query/kinds/categories/scopes/limit/includeSensitive/includeEvidence`。
  - `memory_read` 保留兼容，内部复用 `memory_search` 执行逻辑。
  - 返回结果包含 `scope`、`score`、`scoreBreakdown`、可选 `evidence`。
  - 写入后同时失效 `memory_read` 与 `memory_search` tool cache。
- 更新 `apps/web/src/lib/agent/tools/definitions.ts` 与 `registry.ts`：
  - 新增 `memory_search` tool definition。
  - `memory_read` 标记为兼容旧版读取工具。
  - Agent 默认可用工具包含 `memory_search` 和 `memory_read`。
- 更新 `apps/web/src/app/api/agent/stream/route.ts`：
  - 将 `buildMemoryContext` 改为 `buildInstructionMemoryContext`。
  - 请求开始时只自动注入 `kind=instruction`、`status=active`、`privacy!=sensitive`、user scope 的 compact memory。
  - 注入内容控制在约 1200 字符以内。
  - semantic/episodic/procedural memory 不再自动塞入 system prompt，需要通过 `memory_search` tool 获取。
  - `run_start` trace metadata 记录 `injectedMemoryIds`。
- 更新 `apps/web/src/lib/agent/react-loop.ts`：
  - tool_end trace metadata 记录 memory tool 返回的 `memoryIds`。
- 更新前端展示：
  - `ToolCallCard` 支持 `memory_search` 名称和 memory score 展示。
  - `MessageList` 将 `memory_search` 归类为只读工具。
  - `packages/business/i18n/zh-CN.json` 与 `en.json` 增加 `memorySearchTool` 文案。
- 更新测试：
  - `apps/web/src/lib/agent/__tests__/tools.test.ts` 覆盖 `memory_search` 注册、参数契约、过滤与 `scoreBreakdown`。
  - 新增 `packages/ai/__tests__/memory-retrieval.test.ts`，验证读路径不 upsert，并验证 fallback 输出 scope-aware `scoreBreakdown`。

## 已知风险

- 移除读路径 upsert 后，历史未同步到 Qdrant 的 memory 会更多走 fallback，这是 M24 的预期取舍。
- `memory_search` 已支持 scope/kind/category 过滤，但当前 scope 来源仍以 user scope 和 current document scope 为主，project/module/path scope 需要后续产品链路补充。
- `scoreBreakdown` 是轻量规则版本，暂未引入 LLM rerank 或真实 eval 权重调优。
- compact instruction memory 当前只注入 user scope，后续如需要 project/document instruction memory，需要先完善写入和作用域来源。
