# RAG 检索策略可视化优化

## 本轮目标

- 在 AI Chat 的 tool 可视化中展示 `knowledge_search` 实际使用的检索策略。
- 补强 `knowledge_search` 的 description，让 LLM 更清楚何时选择 `fast` / `balanced` / `deep`。

## 已完成变更

- 更新 `apps/web/src/lib/agent/tools/definitions.ts`：
  - `knowledge_search` description 增加策略选择规则。
  - 简单事实查询默认提示使用 `balanced`。
  - 复杂研究、跨文档归纳、需要更高召回的问题提示使用 `deep`。
  - 极低延迟场景提示使用 `fast`。
  - 用户显式要求某种策略时优先遵循用户要求。
- 更新 `apps/web/src/components/ai-chat/ToolCallCard.tsx`：
  - 折叠摘要中展示检索策略和引用文档数量。
  - 展开详情中展示检索策略。
  - 展开详情中展示 semantic / keyword / metadata / fused 召回统计。
  - 单条文档结果右侧展示命中的召回来源。
- 更新 `apps/web/src/components/ai-chat/types.ts`：
  - 为 `KnowledgeSearchDoc` 补充 `sources` 和 `metadata` 字段。
- 更新 `packages/business/i18n/zh-CN.json` 和 `packages/business/i18n/en.json`：
  - 新增检索策略和召回统计的中英文文案。

## 验证

- `GetDiagnostics`：
  - `ToolCallCard.tsx`：无诊断错误。
  - `types.ts`：无诊断错误。
  - `definitions.ts`：无新增错误，仅存在既有 `Serp` 拼写提示。
- `pnpm --filter @notion/web test -- --run src/lib/agent/__tests__/tools.test.ts`：通过，5 个测试文件，60 个用例。
- `pnpm --filter @notion/web typecheck`：失败，原因是 `@notion/web` 没有 `typecheck` 脚本。
- `pnpm --filter @notion/web exec tsc --noEmit`：通过。

## 已知缺口

- tool 运行中阶段仍主要展示工具名；策略在 tool result 返回后展示。
- 当前只展示融合后的 topK 结果，不展示被裁剪的召回候选。
- 暂未新增专门的 UI 组件单测。

## 下一步建议

1. 如果要更强可观测性，后续可在 tool-call arguments 流式解析后，在运行中也显示 LLM 选择的 `strategy`。
2. 增加 `retrievalTrace`，展示各路召回耗时、降级原因和融合前候选数量。
3. 补 `ToolCallCard` 的前端单测，覆盖 `fast` / `balanced` / `deep` 展示。
