# Phase 5 P1：MCP 写文档输出体验收口

## 目标

复查并优化 MCP 写文档工具的 dry-run 文案、`structuredContent` 和错误输出，确保 Agent 默认安全、可解释、可解析。

## 已完成

- 更新 `packages/my-notion-cli/src/mcp/server.ts`：
  - `toToolResult` 支持 human-readable message + JSON fallback。
  - 新增 `toErrorToolResult`，统一 MCP tool 错误输出。
  - 所有 MCP document tool 都捕获错误并返回 `isError: true`。
  - 错误结果包含 `structuredContent.error.message`、`status`、`code`、`requestId`。
  - `my_notion_docs_create` dry-run 返回 `confirmationRequired: true` 和 no-write message。
  - `my_notion_docs_update` dry-run 返回 `confirmationRequired: true` 和 no-write message。
- 更新 `scripts/e2e-my-notion-mcp.mjs`：
  - 断言 create dry-run 的 `dryRun`、`confirmationRequired`、no-write message 和 text fallback。
  - 断言 update dry-run 的 `dryRun`、`confirmationRequired` 和 no-write message。
  - 断言真实 update 错误返回 `isError: true`、结构化错误和可读 text fallback。
- 更新 `packages/my-notion-skills/my-notion-mcp/SKILL.md`：
  - 补充 dry-run 输出契约。
  - 补充 MCP error result 契约。
- 更新 `docs/my-notion-cli-release-checklist.md`：
  - 增加 MCP dry-run 和错误输出发布检查项。
- 更新架构文档和 `.trae` plan：
  - Phase 5 P1 的 MCP 输出体验复查项标记完成。
