# BlockNote AI 编辑器重构

## 完成的改动

### Step 1：编辑器 AI 加限流
- 修改 `app/api/editor-ai/streamText/route.ts`：复用 `checkRateLimit`，key 使用 `editor-ai:${userId}` 前缀与侧边栏 Agent 隔离

### Step 2：修复 DashScope 兼容性
- 删除显式 `tool_choice: "auto"`，规避 `enable_thinking` + `tool_choice` 冲突导致的 400 错误

### Step 3：统一图标风格
- 修改 `packages/ai/utils/custom-ai-menu-items.ts`：emoji 图标替换为 Remix Icon 名称字符串
- 修改 `apps/web/src/components/editor/customAIMenuItems.tsx`：新增 `ICON_MAP` 将名称映射为 `react-icons/ri` 组件，渲染 `<Icon size={18} />` 与 BlockNote 默认菜单一致

### Step 4：拆分 route.ts
- 新增 `packages/ai/server/editor-ai/inject-document-state.ts`：`injectDocumentStateMessages`
- 新增 `packages/ai/server/editor-ai/convert-openai-messages.ts`：`convertToOpenAIMessages` + `toolDefinitionsToOpenAITools` + `extractTextContent`
- 新增 `packages/ai/server/editor-ai/index.ts`：聚合导出
- 修改 `packages/ai/package.json`：新增 `./server/editor-ai` export
- 重写 `route.ts`：314 行 → 186 行，仅保留路由逻辑和流处理

### Step 5：修复 autoSubmit
- 修改 `customAIMenuItems.tsx`：`toItem` 闭包捕获 `editor`，`autoSubmit=true` 时调用 `editor.invokeAI()` 直接执行，`autoSubmit=false` 时仅 `setPrompt`

### Step 6：抽取 Editor.tsx 职责
- 新增 `apps/web/src/components/editor/useEditorAITransport.ts`：transport 构建 + authFetch
- 新增 `apps/web/src/components/editor/editor-locale.ts`：`getBlockNoteDictionary` + `getAILocaleDict`
- 重写 `Editor.tsx`：移除 transport/locale 内联逻辑，import 抽取后的模块

### Step 7：类型安全
- 新增 `packages/ai/utils/extract-text.ts`：`extractTextFromDocument` 使用 `BlockNode` 类型替代 `any`
- 修改 `packages/ai/utils/index.ts`：从独立文件 re-export

- `pnpm build`: ✅
