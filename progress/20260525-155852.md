# PAT Token 撤销与 CLI E2E 自动清理

## 目标

补齐 PAT Token 的 list/revoke API，并让 CLI 端到端测试脚本在结束时自动撤销本次生成的测试 PAT，避免 dev 环境长期残留可用 token。

## 已完成

- 在 `@notion/convex/cli` 中新增 `listApiTokenRecords`：
  - 仅允许登录态用户列出自己的 PAT 元信息。
  - 不返回 token 明文或 hash。
- 在 `@notion/convex/cli` 中新增 `revokeApiTokenRecord`：
  - 仅允许 token 所有者撤销。
  - 通过写入 `revokedAt` 软撤销，保留审计信息。
- 扩展 `POST /api/cli/tokens` 所在 route：
  - `GET /api/cli/tokens`：列出当前用户 token。
  - `DELETE /api/cli/tokens?tokenId=...`：撤销指定 token。
- 更新 `scripts/e2e-my-notion-cli.mjs`：
  - 捕获 seed token 返回的 `tokenId`。
  - 在正常结束时撤销测试 PAT。
  - 在异常场景下通过 `finally` 尽量撤销测试 PAT。
  - 撤销后再次执行 `auth status`，确认 revoked token 已无法认证。

## 已知风险

- E2E 仍会保留测试文档，目前只清理 token；后续可补文档归档/删除能力。
- Web token 管理 API 已具备 list/revoke，但尚未接入 UI。

## 下一步建议

- 给 CLI 增加 `auth tokens list/revoke` 或 `tokens list/revoke` 命令，方便用户从终端管理 PAT。
- 为 `my-notion-skills` 编写首版技能说明，让 Agent 自动使用 CLI 生成和更新文档。
