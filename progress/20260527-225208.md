# Memory Review 缺口收口

## 本轮目标

- 修复 Memory Review UI 的三个已知缺口：
  - 停用操作增加二次确认。
  - Review UI 支持手动新增记忆。
  - `memory_write` 聊天卡片的保存/取消状态持久化到历史消息。

## 已完成变更

- 新增 `updateToolResultState` Convex mutation：
  - 仅允许当前用户更新自己 assistant 消息里的 `memory_write` tool result。
  - 支持将指定 tool result 标记为 `saved` 或 `cancelled`。
  - 保存状态会写入 `memoryWriteStatus`，保存成功时额外写入 `savedMemoryId`。
- 更新 `ToolCallCard`：
  - `memory_write` 保存成功后先写入长期记忆，再把 tool result 状态持久化为 `saved`。
  - 取消保存时把 tool result 状态持久化为 `cancelled`。
  - 刷新历史消息后不再重复展示确认按钮。
- 更新 `MessageList`：
  - 对已持久化的 assistant 消息，将 `messageId` 传给 `ToolCallCard`。
- 更新 `MemoryReviewPage`：
  - 新增“新增记忆”表单，支持手动创建 `manual` 来源记忆。
  - 抽出复用的 `MemoryForm`，新增和编辑共用同一套表单。
  - 停用操作接入 `AlertDialog` 二次确认。
- 更新中英文 i18n：
  - 新增创建记忆、停用确认、确认停用等文案。

## 已知缺口

- Memory 仍未接 embedding / Qdrant，Review UI 搜索仍是结构化轻量匹配。
- Agent Loop 尚未自动注入相关 memory，仍主要依赖 LLM 自主调用 `memory_read`。

## 下一步建议

1. 将 active memory 自动注入 Agent loop 的系统上下文或首轮 tool context。
2. 为 memory 增加 embedding 索引，提升跨表达检索能力。
3. 给 Memory Review UI 补组件级测试，覆盖新增、编辑、停用确认和空状态。
