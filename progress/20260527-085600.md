## 已知风险

- 第一版 keyword recall 如果只做轻量 token match，效果可能弱于正式 BM25/全文索引，但实现成本最低，适合作为 MVP。
- Metadata recall 需要依赖文档标题、标签、最近编辑时间、chunkIndex 等索引字段，后续实现时需要确认当前 Qdrant payload 与 Convex 文档元数据是否足够。
- 融合排序需要避免单一路召回分数支配结果，建议优先考虑 RRF，再根据实际效果调权重。

## 下一步建议

1. 实现 `retrieveKnowledge(options)`，其中 `fast` 保留现有 Qdrant 向量检索行为。
2. 实现 `balanced` 的 semantic recall + keyword recall + metadata recall + fusion/dedup。
3. 改造 `knowledge_search` tool，让默认策略使用 `balanced`。
4. 增加 focused tests，覆盖精确关键词、语义改写、metadata 过滤和去重融合。
