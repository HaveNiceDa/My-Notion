# PAT 创建流程 Playwright 测试

## 目标

为 Web 设置页 PAT 创建流程补充 Playwright 覆盖，重点验证 scope 选择和过期时间校验逻辑。

## 已完成

- 新增 `tests/web/pat-settings.spec.ts`：
  - 使用 `page.route("**/api/cli/tokens")` mock `GET` 与 `POST`。
  - 取消选择 `docs:read` 和 `docs:write` 后点击创建，断言不会发送 `POST`。
  - 只选择 `docs:read` 并填入过去时间后点击创建，断言不会发送 `POST`。
  - 填入未来过期时间后创建，断言 `POST` body 包含：
    - `name: "Playwright PAT"`
    - `scopes: ["docs:read"]`
    - `expiresAt` 为未来毫秒时间戳
  - 验证创建成功后出现一次性明文 token 面板。
  - 验证复制按钮、勾选“我已保存”和隐藏 token 的流程。
- 更新 `playwright.config.ts`：
  - `pat-settings.spec.ts` 只归入 `chromium-authenticated` project。
  - 未登录的 `chromium` project 忽略该测试，避免被保护路由重定向影响。

## 验证结果

- `pnpm --filter @notion/web exec tsc --noEmit`：通过。
- `pnpm --filter @notion/web lint`：通过，仍有 7 个历史 warning，非本次新增。
- `CI= pnpm exec playwright test tests/web/pat-settings.spec.ts --list`：通过，确认测试挂在 `chromium-authenticated` project。

## 未完成验证

- `CI= pnpm exec playwright test tests/web/pat-settings.spec.ts --project chromium-authenticated` 未完整运行：
  - 当前环境缺少 `CLERK_SECRET_KEY`，`auth-setup` 无法生成登录态。
- `CI= pnpm exec playwright test tests/web/pat-settings.spec.ts --project chromium-authenticated --no-deps` 也未通过：
  - 当前 `.auth/storage-state.json` 已失效，页面被重定向到营销页。

## 下一步建议

- 在具备 `CLERK_SECRET_KEY` 或有效 `.auth/storage-state.json` 的环境中运行：

```bash
CI= pnpm exec playwright test tests/web/pat-settings.spec.ts --project chromium-authenticated
```

- 若希望该 UI 测试在 CI 中运行，需要给 CI 配置 Clerk 测试凭据或增加更稳定的测试认证方案。
