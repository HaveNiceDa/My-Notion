# 只读 Tool 跨请求缓存

## 本轮目标

- 按 AI Chat 重构计划的下一步建议，实现 `Tool 结果缓存`。
- 将已有的单次 Agent run 内去重扩展为只读 Tool 跨请求 5 分钟缓存。
- 补测试覆盖缓存命中、TTL、写类 Tool 不缓存和用户隔离。

## 已完成变更

- 新增 `apps/web/src/lib/agent/tool-result-cache.ts`：
  - 支持 5 分钟 TTL。
  - 支持最大 200 条的进程内 LRU 淘汰。
  - 缓存 key 包含 `userId`、`toolName`、规范化后的 arguments、当前文档 `id/title/content`。
  - 只允许只读 Tool 进入跨请求缓存：`knowledge_search`、`web_search`、`web_extract`、`document_search`、`document_read`、`memory_read`。
  - 写类 Tool 不进入跨请求缓存：`document_write`、`document_update`、`memory_write`。
  - 提供 `clearToolResultCache()`，便于测试隔离。
- 更新 `apps/web/src/lib/agent/react-loop.ts`：
  - 保留单次 Agent run 内 `toolResultCache`，继续处理同名同参重复调用。
  - 在执行真实 Tool 前先查询跨请求只读缓存。
  - 跨请求缓存命中时直接向前端发送 `tool-call-result`，并把结果加入 LLM messages。
  - Tool 成功执行后写入跨请求缓存。
  - Tool 返回 `error` 时不写入跨请求缓存，避免短暂失败被缓存放大。
  - trace 增加 `cacheScope: "run" | "shared"`，区分本轮缓存和跨请求缓存。
- 更新 `apps/web/src/lib/agent/__tests__/react-loop.test.ts`：
  - 覆盖跨请求复用只读 Tool 结果。
  - 覆盖不同用户不共享缓存。
  - 覆盖写类 Tool 不进入跨请求缓存。
  - 保留达到 ReAct 工具轮次上限后强制最终回答的原有回归测试。
- 新增 `apps/web/src/lib/agent/__tests__/tool-result-cache.test.ts`：
  - 覆盖参数顺序规范化。
  - 覆盖 5 分钟 TTL 过期。
  - 覆盖用户和当前文档上下文隔离。
  - 覆盖写类 Tool 不写入跨请求缓存。
- 更新 `docs/ai-chat-refactor-plan.md`：
  - 将 `2.4 Tool 结果缓存` 状态更新为完成。
  - 更新建议优先级，下一步前移为 Memory 增强和 RAG 质量补齐。
  - 更新 Agent Loop 当前基线，记录只读 Tool 跨请求缓存能力。

## 缓存边界

- 当前实现是进程内缓存，不跨 serverless 实例、进程重启或多副本共享。
- 当前不缓存错误结果。
- 当前只缓存只读 Tool，不缓存任何会产生写入预览或数据修改的 Tool。
- `document_read` 的缓存 key 包含当前文档内容，前端传入内容变化时会自动避开旧缓存。
- `memory_read` 虽属于只读 Tool，但如果用户刚写入/停用记忆，5 分钟内可能仍命中旧结果；后续做 Memory Qdrant 显式同步时可一并增加记忆写入后的缓存失效。

## 下一步建议

1. 继续做 Memory 增强：显式 Qdrant 同步、embedding 状态字段和 Memory eval。
