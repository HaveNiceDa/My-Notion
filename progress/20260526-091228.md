# Phase 5 P0：E2E 测试文档清理

## 目标

实现 Phase 5 P0 的 E2E 副作用清理：CLI/MCP E2E 在测试结束后不仅撤销临时 PAT，也软归档测试过程中创建的文档，避免 dev Convex 部署持续累积测试数据。

## 已完成

- 新增机器 API 文档归档能力：
  - `DELETE /cli/v1/documents/:id`
  - 内部调用 `cli:archiveCliDocument`
  - 采用软归档 `isArchived: true`，不做硬删除。
- 新增 CLI 命令：
  - `my-notion docs archive --id <documentId> --format json`
  - 复用现有 PAT、`docs:write` scope 和机器 API client。
- 更新 CLI E2E：
  - 从 9 步扩展为 10 步。
  - 在 `tokens revoke-current` 前执行 `docs archive`。
  - 归档后再次搜索，断言测试文档不再出现在普通 search 结果中。
  - 在 `finally` 中增加兜底归档逻辑，随后再撤销 PAT。
- 更新 MCP E2E：
  - 从 8 步扩展为 9 步。
  - MCP 完成 create/fetch/update/search 后关闭 STDIO server。
  - 使用 CLI `docs archive` 归档 MCP 创建的测试文档。
  - 在 `finally` 中增加兜底归档逻辑，随后再撤销 PAT。
- 更新 Skills 文档：
  - `my-notion-docs` 增加 archive 使用说明。
  - `references/cli-commands.md` 增加 `docs archive` 命令、输出字段和 E2E cleanup 说明。
  - 已运行 `pnpm sync:skills` 同步 `.trae/skills`。
