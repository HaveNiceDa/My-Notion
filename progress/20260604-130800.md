# Agent 记忆 Schema 精简

## 目标

继续清理 Agent 记忆系统的复杂心智，将 Convex schema 和相关类型也收敛到最新三类基础记忆模型，避免数据库层继续保留通用 Agent Memory 治理字段。

## 完成内容

- 精简 `agentMemories` schema，只保留：
  - `userId`
  - `type`
  - `content`
  - `source`
  - `reason`
  - `summary`
  - `tags`
  - `evidenceText`
  - `confidence`
  - `status`
  - `createdAt`
  - `updatedAt`
- 删除 schema 中的复杂字段：
  - `kind/category`
  - `scopeLevel/scopeKey`
  - `importance/stability/privacy`
  - `supersededBy/supersedes/conflictsWith`
  - `expiresAt/reviewDueAt/lastUsedAt/usageCount`
  - `embeddingRef/embeddingStatus/embeddingUpdatedAt/embeddingError/embeddingRetryCount/nextEmbeddingRetryAt`
  - `evidenceConversationId/evidenceMessageId/evidenceDocumentId/evidenceToolCallId/evidenceUrl`
- 删除复杂索引，只保留：
  - `by_user_and_status`
  - `by_user_type_and_status`
  - `by_user_and_updatedAt`
- 精简 `agentMemories/model.ts`，移除 kind/scope/privacy/embedding/stability 默认派生层。
- 精简 Convex 读写逻辑，写入和列表输出不再处理复杂治理字段。
- 精简 AI server memory ranking，删除 scope/importance/usage/staleness/scoreBreakdown 评分维度。
- 精简 Qdrant memory payload，只索引 type/content/summary/reason/source/confidence。
- 精简 memory eval fixture/evaluator，保留基础检索评估，冲突/复杂写入 eval 置为空实现。
- 删除旧 `agent-memories-model.test.ts`，因为复杂默认派生层已移除。

## 注意

- 本次直接删除 schema 字段，若已有 Convex 数据中存在旧字段，部署前需要确认目标环境是否允许直接收窄 schema，或先清理旧数据。
- 当前代码心智已统一为三类基础记忆：用户偏好、项目规则、最近决策。
