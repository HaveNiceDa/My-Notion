# 2026-05-27 CLI Tokens and Dialog Accessibility Fix

## 背景

- 本地运行时 Settings 弹窗触发 Radix `DialogContent` 缺少 description 的无障碍警告。
- `/api/cli/tokens` 在 Clerk `getToken` 或 Convex 查询遇到 `fetch failed` / `ETIMEDOUT` 时返回 500，影响设置面板加载体验。

## 变更

- 为通用 `AlertDialogContent` 增加 `aria-describedby` fallback 和隐藏描述，避免调用方遗漏 `AlertDialogDescription` 时触发 Radix 警告。
- 为 Settings 主弹窗补充隐藏的 `DialogDescription`。
- 为 CLI token API 增加网络不可用识别：
  - `GET /api/cli/tokens` 在 token 服务临时不可用时降级返回空列表和 warning，避免设置页直接报 500。
  - `POST` / `DELETE` 属于写操作，服务不可用时返回 `503 SERVICE_UNAVAILABLE`，避免误报成功。
- 修复 `AuthenticatedConvexClientResult` union narrowing，使用 `in` operator 保持类型安全。

## 验证

- `pnpm --filter @notion/web exec tsc --noEmit`
- `pnpm --filter @notion/web lint`
- `pnpm --filter @notion/web build`

## 备注

- lint 仍存在既有 warning：若干 `<img>` 使用和一个 `react-hooks/exhaustive-deps` 提示，本轮未扩大范围处理。
- build 仍提示 Sentry 未配置 `authToken`，不影响本地构建通过。
