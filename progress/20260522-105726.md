# AI Chat 侧边栏 Notion 风格补齐

## 完成的改动

- 将 AI Chat 右侧面板顶部改为常驻操作区：首页态和对话态都保留历史对话、新建、固定/取消固定、关闭入口。
- 历史对话从内联折叠区调整为顶部浮层，并补充搜索输入，避免挤压当前首页或对话内容。
- 新增侧边栏固定状态 `panelPinned`：固定时占用主布局宽度，取消固定时以右侧浮层覆盖。
- 补齐 Notion 风格首页：标题提示、总结页面、翻译页面、深度剖析、创建任务跟踪器、额度提示、开题记录卡片。
- 补齐 `zh-CN`、`zh-TW`、`en` 的 AI 侧边栏新增文案。
- 修复 `layout.tsx` 中 `handleSave` 在 Hook 中声明顺序导致的 lint error。
- 已关闭占用 `3000` 端口的本地进程。

## 验证命令和结果

- `lsof -ti tcp:3000 | xargs kill -9`: ✅
- `pnpm --filter @notion/web exec tsc --noEmit`: ✅
- `pnpm --filter @notion/web lint`: ✅ 通过，有 6 个既有 warning
- `pnpm --filter @notion/web build`: ✅ 通过，有 Sentry auth token 和 edge runtime 常规 warning

## 已知缺口或风险

- 首页快捷动作目前会把预置 prompt 填入输入框，尚未接入当前文档内容读取 tool。
- 固定状态尚未持久化到 localStorage，刷新后默认固定。
- lint 仍存在既有 warning：Navigation 的 effect setState、MessageList 图片优化、useAIChat effect 依赖等。

## 下一步建议

- 接入 `document_read` tool 后，让“总结/翻译/深度剖析此页面”真正读取当前文档内容。
- 将 `panelPinned` 持久化，保持用户上次侧边栏布局偏好。
- 继续推进 `/api/agent/stream` 与 `knowledge_search` tool 抽离。
