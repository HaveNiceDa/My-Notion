# MCP STDIO E2E 脚本与注释补强

## 目标

将 MCP STDIO smoke 流程固化为可复用 E2E 脚本，并为本次新增的 MCP server、MCP 命令入口和 E2E 脚本补充维护注释。

## 已完成

- 新增 `scripts/e2e-my-notion-mcp.mjs`：
  - 构建 `@notion/my-notion-cli`。
  - 为临时测试用户生成 `mnt_` 前缀 PAT。
  - 使用 Convex CLI 将 PAT hash 注入 dev 部署。
  - 通过 `node packages/my-notion-cli/dist/index.js mcp serve --transport stdio` 启动 MCP server。
  - 使用 line-delimited JSON-RPC 直接验证 STDIO transport。
  - 覆盖 `initialize`、`tools/list`、`tools/call`。
  - 验证 4 个 MCP tools 存在：
    - `my_notion_docs_search`
    - `my_notion_docs_fetch`
    - `my_notion_docs_create`
    - `my_notion_docs_update`
  - 验证 create/update 的 `dryRun: true` 不触发真实写入。
  - 验证真实 `dryRun: false` 创建、读取、追加更新和搜索链路。
  - 在 `finally` 中撤销测试 PAT，并删除临时 `HOME`。
- 根 `package.json` 新增脚本：
  - `pnpm e2e:mcp`
- 为 MCP 相关代码补充注释：
  - `packages/my-notion-cli/src/mcp/server.ts`
    - 说明 `structuredContent` 与 text fallback 的兼容原因。
    - 说明 dry-run 不触发 API，只返回 document-shaped preview。
    - 说明 read-only tools 的无副作用边界。
    - 说明 write tools 默认 dry-run 和显式授权要求。
    - 说明 STDIO 是当前 MVP 唯一支持 transport。
  - `packages/my-notion-cli/src/commands/mcp.ts`
    - 说明拒绝未知 transport 是为了避免暗示未实现的鉴权语义。
  - `scripts/e2e-my-notion-mcp.mjs`
    - 说明脚本通过真实 STDIO JSON-RPC 启动 CLI。
    - 说明 line-delimited JSON-RPC framing。
    - 说明测试失败时也必须撤销 PAT。

## 已知风险

- MCP E2E 当前仍会保留测试文档，只清理测试 PAT。
- 脚本依赖 dev Convex 部署和 `NEXT_PUBLIC_CONVEX_URL` 或 `MY_NOTION_E2E_API_URL`。
- 当前只验证 STDIO MCP，不覆盖 HTTP MCP 或 OAuth。
