# Metadata 索引增强与召回升级

## 本次目标

继续增强 Hybrid Retrieval 的 metadata 召回能力：索引时补充 `updatedAt`、`tags`、层级标题、chunk 邻近摘要，并升级 `metadataSearch()` 的召回打分逻辑。

## 已完成

- 扩展 `DocumentUpdateParams`，支持可选 `updatedAt`、`tags`、`documentPath`。
- `/api/rag-documents` 透传 metadata 参数到 `@notion/ai/server` 的 `updateDocument()`。
- 文档编辑页触发 RAG 更新时传入当前更新时间。
- 发布弹窗添加知识库时传入 `lastEditedTime` 作为 `updatedAt`。
- `QdrantVectorStoreWrapper.updateDocument()` 索引 chunk 时新增 metadata：
  - `updatedAt`
  - `tags`
  - `documentPath`
  - `headingPath`
  - `headings`
  - `neighborSummary`
- 支持从 BlockNote heading 块提取层级标题，并从内容中的 `#tag` 提取内联标签。
- Qdrant payload index 新增 `metadata.updatedAt`、`metadata.tags`、`metadata.headingPath`、`metadata.documentPath`。
- 升级 `metadataSearch()` 打分逻辑，支持标题、documentId、tags、headingPath、headings、documentPath、neighborSummary 和 recency boost。
- 新增 `packages/ai/__tests__/qdrant-metadata.test.ts`，覆盖 tag、heading、neighbor summary 的 metadata 召回。

## 已知风险

- Convex `documents` 表目前没有正式 `tags` 字段；本轮先支持外部传入 tags，并从文档正文中的 `#tag` 提取内联标签。
- `headingPath` 当前按文档级 heading 聚合，不是严格的每个 chunk 所在层级；后续可以改为按 BlockNote block 到 chunk 的映射生成更精确的 chunk-level heading path。
- `neighborSummary` 当前由前后 chunk 截断拼接生成，适合 MVP；后续可替换为 LLM/规则摘要。

## 下一步建议

1. 实现 `deep` 策略的 query rewrite，生成原问题、关键词版、语义扩展版。
2. 将 `deep` 接入 multi-query search，对每个 query 执行 semantic + keyword 召回后统一融合。
3. 开始记录 retrieval trace，为后续 harness / eval 回放做准备。
