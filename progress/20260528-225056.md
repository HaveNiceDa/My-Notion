## Context Packing 说明

- Context Packing 的作用是把检索结果从“散碎 chunk 列表”整理成“更适合喂给 LLM 的上下文”。
- 它会先按文档分组，再把同一文档里相邻的 chunk 合并起来，避免模型看到多个割裂片段。
- 它还会按 token budget 裁剪上下文，防止一次检索返回太多内容挤占对话上下文。
- 直接收益：
  - 回答更连贯，减少断章取义。
  - 引用更清晰，能知道哪些 chunk 被合并。
  - 上下文长度可控，避免超 token 或挤掉用户问题。
  - 后续 Citation Quality / eval 可以基于 packed metadata 做可解释分析。
