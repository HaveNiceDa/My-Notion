# Memory 缓存失效与 Qdrant 显式同步

## 本轮目标

- 在 Agent Tool 结果缓存落地后，补齐 Memory 写入、编辑、停用后的缓存一致性。
- 将 Memory 的 Qdrant 同步从读取时懒 upsert 增强为写入路径显式同步。
- 保持 Convex 结构化源记录为可信来源，Qdrant 只作为召回索引。

## 已完成变更

- `apps/web/src/lib/agent/tool-result-cache.ts`：
  - 新增 `invalidateToolResultCache(filter)`。
  - 支持按 `userId` 和 `toolNames` 精确清理跨请求 Tool 缓存。
  - 用于 Memory 变更后只清理当前用户的 `memory_read`，避免影响其他只读 Tool 缓存。
- `packages/ai/server/memory.ts`：
  - 新增 `syncAgentMemory({ userId, memory })`，显式 upsert 单条 Memory 到 Qdrant。
  - 新增 `deleteAgentMemoryIndex({ userId, memoryId })`，删除 `memory:<memoryId>` 对应向量 chunk。
  - 保留 `retrieveRelevantMemories` 内的懒 upsert，作为显式同步失败或历史数据未同步时的兜底。
- `apps/web/src/app/api/agent/memories/sync/route.ts`：
  - 新增 Memory 同步 API。
  - `upsert`：先清理当前用户 `memory_read` 缓存，再同步单条 Memory 到 Qdrant。
  - `delete`：先清理当前用户 `memory_read` 缓存，再删除对应 Qdrant memory chunk。
  - Qdrant 不可用时返回 `success: true + warning`，不阻断 Convex 源记录写入。
- `apps/web/src/lib/agent/memory-sync-client.ts`：
  - 新增前端轻量同步 helper：`syncMemoryIndex` / `deleteMemoryIndex`。
- `apps/web/src/components/ai-chat/MemoryReviewPage.tsx`：
  - 手动新增 Memory 成功后同步 Qdrant。
  - 编辑 Memory 成功后同步 Qdrant。
  - 停用 Memory 成功后删除 Qdrant 索引。
  - 同步失败仅 console warning，不回滚 Convex 源记录。
- `apps/web/src/components/ai-chat/ToolCallCard.tsx`：
  - Agent 提议 Memory 经用户确认保存后，同步 Qdrant 并清理 `memory_read` 缓存。
- `apps/web/src/lib/agent/tools/memory.ts`：
  - `memory_write` 在 `dryRun=false` 真实写入后，清理当前用户 `memory_read` 缓存并显式同步 Qdrant。
  - 同步失败时返回 warning，但不影响 Memory 已保存结果。
- `packages/convex/agentMemories/logic/update.ts`：
  - `updateAgentMemory` 返回值补充 `source`，方便前端编辑后直接同步 Qdrant。
- 测试：
  - `tool-result-cache.test.ts` 增加按用户和 tool name 失效 `memory_read` 缓存的覆盖。
  - `tools.test.ts` 增加 `memory_write dryRun=false` 后调用 `syncAgentMemory` 的断言。

## 一致性策略

- Convex 仍是 Memory 的结构化可信源。
- Qdrant 同步失败不阻断用户写入，避免因为向量服务不可用导致 Memory Review 或确认保存失败。
- `memory_read` 缓存会在写入/编辑/停用后立即按用户失效，避免 5 分钟 TTL 内读到旧 Memory 列表。
- 停用 Memory 会删除对应 `memory:<memoryId>` 向量 chunk；即使删除失败，后续 `memory_read` 的 active memory id 过滤也会避免返回已停用 Memory。

## 已知风险

- Qdrant 同步失败目前只做 warning 降级，尚未把失败状态写回 Convex，也没有后台重试队列。
- `embeddingRef` / embedding 同步状态仍未在 Memory Review UI 展示。
- Memory E2E / eval 仍未补齐。

## 下一步建议

1. 补 Memory embedding 状态：在 Convex 记录中展示 `synced` / `pending` / `failed`。
2. 增加 Qdrant 同步失败重试队列，避免短暂不可用导致长期索引漂移。
3. 补 Memory E2E / eval：覆盖创建、编辑、停用后 `memory_read` 不读旧数据。
4. 进入 RAG Context Packing 与 Citation Quality，提升知识检索答案质量。
