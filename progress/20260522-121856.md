# 文档顶部栏归属主内容区

## 完成的改动

- 新增 `MainContentNavbar`，将文档顶部栏渲染到 `main` 内部的 `absolute left-0 right-0 top-0` 容器。
- 从 `Navigation` 中移除顶部栏渲染、`navbarRef` 和手动维护 `left/width` 的逻辑。
- `Navigation` 现在只负责左侧导航栏自身的折叠、展开、拖拽宽度，不再影响文档内容区或 AI 侧边栏。
- `layout.tsx` 将 `main` 改为 `relative`，让顶部栏的绝对定位边界天然限制在文档内容区内。
- AI 侧边栏层级从临时 `z-[100000]` 降回普通层级，避免继续依赖层级压制解决布局覆盖。
- 修复 `Navigation` 中移动端折叠逻辑的 lint warning：effect 内只更新共享导航 store，不再直接调用带 React setState 的 `collapse/resetWidth`。

## 验证命令和结果

- `pnpm --filter @notion/web exec tsc --noEmit`: ✅
- `pnpm --filter @notion/web lint`: ✅ 通过，有 4 个既有 warning
- `pnpm --filter @notion/web build`: ✅ 通过，有 Sentry auth token 和 edge runtime 常规 warning

## 已知缺口或风险

- 左侧导航栏拖拽宽度目前仍通过 DOM style 保存到当前会话，未持久化。
- `MainContentNavbar` 与 `Navigation` 通过 `useNavigation` store 协调折叠状态，后续如扩展导航宽度持久化，可继续沉淀到 store。

## 下一步建议

- 将左侧导航栏宽度和 AI 侧边栏固定状态一起持久化。
- 清理文档页 `flushRagUpdates/flushUpdates` effect 依赖 warning。
