# Agent 性能 Trace 基础监控

## 本轮目标

- 打通 AI Chat Agent 的基础性能监控，为 ReAct / LLM / tool 执行提供统一 trace 和耗时日志。
- 先实现本地可用的结构化 console trace，并预留后续接 Sentry span / Harness replay 的 sink。

## 已完成变更

- 新增 `apps/web/src/lib/agent/trace.ts`：
  - 定义 `AgentTraceEvent`、`AgentTraceEventType` 和 `AgentTracer`。
  - 支持统一 `traceId`、`elapsedMs`、`durationMs`、基础 metadata 合并。
  - 提供 `summarizeForTrace`，对 tool arguments 做摘要、截断和敏感字段脱敏。
- `/api/agent/stream` 请求级 trace：
  - 记录 `run_start` / `run_end` / `run_error`。
  - metadata 包含 route、conversationId、model、enableThinking、toolNames、memoryInjected、message count。
- ReAct loop trace：
  - 记录 `react_iteration_start` / `react_iteration_end`。
  - 记录每个 tool 的 `tool_start` / `tool_end` / `tool_error` / `tool_unknown`。
  - tool trace 包含 toolName、toolCallId、argsSummary、resultLength、recoverable、sourceCount。
- LLM stream trace：
  - 记录 `llm_start`、`llm_first_chunk`、`llm_end`、`llm_error`。
  - 记录首 chunk 延迟、总耗时、text/reasoning delta 数量、tool call 数量。
- 更新 `docs/ai-chat-refactor-plan.md`：
  - `3.2 Agent 性能监控` 标记为基础完成。
  - 下一步优先级调整为 `web_extract` + `document_search`。
  - 将后续 Sentry/Harness sink 作为增强项保留。

- `pnpm --filter @notion/web build`：通过；仅 Sentry auth token 缺失 warning。
- `GetDiagnostics`：无诊断问题。

## 已知风险

- 当前 trace sink 只输出结构化 console，不持久化到数据库，也未真正创建 Sentry span。
- tool arguments 只做通用字段脱敏，后续如果新增敏感参数，应扩展脱敏 key。
- `run_end` 统计的是流式执行主体耗时，不包含鉴权、memory context 构建和 context compression 的前置耗时。

## 下一步建议

1. 继续做 `web_extract` 和 `document_search`，扩展两个高收益只读 tool。
2. 结合真实对话日志观察 tool 选择、LLM 首 chunk 和 tool 耗时分布。
3. 后续将 `AgentTracer` sink 接入 Sentry span 或持久化 trace 表，为 Harness replay 提供数据源。
