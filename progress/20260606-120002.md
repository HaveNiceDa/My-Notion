# 续跑一致性收口

## 目标

- 按优先级补齐 assistant 消息原地更新，避免续跑完成后历史消息仍停留在中断内容。
- 为仍在运行中的 Agent run 提供受控接管能力，让“继续生成”不只处理 completed/failed run。

## 已完成

- 新增 `updateAssistantMessage` Convex mutation，仅允许会话 owner 更新 assistant 消息内容。
- `useAIChatPersistence` 增加 `updateAssistantMessage` 封装。
- `resumeLastRun` 完成后，如果 assistant 消息已落库，则原地更新同一条 `aiMessages`；如果仍是临时消息，则保持原有保存新消息逻辑。
- `stream-client` 改为只有收到 `finish` 才触发 `onComplete`，避免 running 接管窗口关闭时误清理 cursor。
- `/api/agent/stream` 对 `running` run 增加受控长轮询接管：resume replay backlog 后继续查询新增 `agentRunEvents` 并向前端续传。
- 长轮询窗口到期但 run 仍未完成时返回 recoverable error，前端保留 cursor，用户可再次点击“继续生成”。
- 补充 `stream-client` 测试覆盖无 `finish` 不误完成、running 接管窗口错误不触发完成。

## 已知缺口

- running run 当前采用长轮询窗口接管，不是真正的实时订阅/推送；窗口到期后需要用户再次点击继续。
- `updateAssistantMessage` 当前更新整段消息 payload，不做字段级 merge；调用方需传入完整 assistant payload。
- Trace Replay UI 和 trace sink 关联仍未实现。
