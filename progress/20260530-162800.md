# 默认 CLI Token 改造与生产 Convex 部署

## 背景

- 线上设置页 `GET /api/cli/tokens` 曾返回 500。
- 用户希望 API Token 管理更简单：登录后默认有一个 token，像密码一样可隐藏/显示/复制，不再要求创建多个 PAT、scope、过期时间和“一次性展示”。

## 已部署

- 生产 Convex deployment：`prod:laudable-albatross-174`。
- 生产 Convex URL：`https://laudable-albatross-174.convex.cloud`。
- 生产 Machine API URL：`https://laudable-albatross-174.convex.site`。
- 已执行并成功：
  - `CONVEX_DEPLOYMENT=prod:laudable-albatross-174 pnpm --filter @notion/web exec convex deploy`

## 代码改造

- `apiTokens` schema 新增 `tokenPlaintext` 可选字段，用于个人版设置页隐藏/显示和复制默认 CLI token。
- Convex CLI 模块新增：
  - `ensureDefaultApiTokenRecord`
  - `resetDefaultApiTokenRecord`
- Web `GET /api/cli/tokens` 会自动 ensure 当前用户的默认 token。
- Web `POST /api/cli/tokens` 支持 `resetDefault: true`，用于重置默认 token 并让旧 token 立即失效。
- 设置页 UI 改为默认 token 卡片，支持显示/隐藏、复制、重置和撤销。
- CLI 默认 Machine API URL 从 dev deployment 修正为生产 `.site` 地址：`https://laudable-albatross-174.convex.site`。

## 后续精简与稳定性修复

- 设置页默认 Token 卡片进一步精简：
  - 移除 prefix badge、长说明、scopes 和 expires 展示。
  - 保留状态、Token 输入框、显示/复制、重置/撤销，以及 created/last used 摘要。
- 修复切换语言时偶发重新请求 token：
  - `loadTokens` 不再依赖 `useTranslations()` 返回的 `t` 函数。
  - locale 切换不会因为翻译函数变化触发 `/api/cli/tokens` 重拉。
- 修复 Clerk 网络错误识别：
  - `ClerkAPIResponseError.errors[]` 现在会被纳入错误文本解析。
  - `unexpected_error / fetch failed / ETIMEDOUT / EHOSTUNREACH` 会被识别为临时不可用并走降级，不再作为 GET 的 `console.error`。
- 避免 `next-intl` 运行时缺 key：
  - 组件不再依赖新增的 `tokenTemporarilyUnavailable` 和 `defaultTokenMeta` key。
  - 改用已有 `failedToLoadTokens`、`createdAt`、`lastUsedAt` 组合，减少动态 messages 缓存导致的 `MISSING_MESSAGE`。
