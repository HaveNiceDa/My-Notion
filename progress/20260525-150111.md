# My-Notion CLI MVP 命令实现

## 目标

实现 `packages/my-notion-cli` 的最小可用命令，直接消费已完成的 Convex HTTP Actions，覆盖认证配置和文档 CRUD 入口。

## 已完成

- 新增轻量参数解析入口，替换原 `src/index.ts` 占位输出。
- 新增 CLI 配置存储，默认写入 `~/.my-notion/config.json`，支持 `MY_NOTION_API_URL` 和 `MY_NOTION_API_TOKEN` 覆盖。
- 新增 fetch-based HTTP client，统一处理 Bearer token、JSON 响应和 API 错误。
- 实现 `auth login`：校验 token 并保存 `apiUrl/token`。
- 实现 `auth status`：读取保存的配置或环境变量，校验当前 token。
- 实现 `docs create/fetch/search/list/update`：
  - `create` 支持 `--title`、`--content`、`--content-file`。
  - `fetch` 支持 `--format markdown` 输出 Markdown 内容。
  - `search/list` 支持 `--limit`。
  - `update` 支持 `--title`、`--content`、`--content-file`、`--mode overwrite|append`。
- 配置 `.gitignore` 忽略 `**/dist`。

## 验证

- `pnpm --filter @notion/my-notion-cli typecheck`：通过。
- `pnpm --filter @notion/my-notion-cli build`：通过。
- `pnpm --filter @notion/my-notion-cli dev -- --help`：通过，CLI help 正常输出。
- `GetDiagnostics`：无诊断错误。
- `git status --short --ignored packages/my-notion-cli .gitignore`：确认 `packages/my-notion-cli/dist/` 已被忽略。

## 已知风险

- 尚未接入真实 PAT 做端到端调用验证；需要用户在 Web 登录态下先通过 `POST /api/cli/tokens` 创建 token。
- 当前参数解析器是轻量实现，不支持复杂短参数、交互式输入或 shell completion。
- token 当前保存到本地 JSON 文件，后续可考虑系统 keychain。
