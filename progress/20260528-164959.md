## 已完成

- Phase 6.1 技术债：
  - `useAIChat.ts` 已拆为状态、流式、持久化和 stream client 组合层。
  - `AIChatPanel.tsx` 已拆出 `ConversationList` 和 `EmptyHome`。
  - Agent 后端测试已落地，当前抽查到 7 个测试文件，覆盖 tools、document-read、context-compression、stream、rate-limiter、react-loop、trace。
  - `AIChatErrorBoundary` 已实现。
  - Agent/stream 长参数列表已重构为结构化 options 对象。
- Phase 6.2 AI 能力：
  - 对话上下文压缩已实现，并纳入测试。
  - 单次 Agent run 内已支持同名同参 tool result cache，用于避免 ReAct 循环内重复执行。
- Phase 6.3 工程化：
  - Agent 基础性能监控已完成，`AgentTracer` 覆盖 run、iteration、LLM 首 chunk / 总耗时、tool 执行耗时与错误。
  - Agent Trace 和 ReAct 明细日志已降噪，默认关闭，按 `AGENT_TRACE_LOG` / `AGENT_DEBUG_LOG` 开启。
  - 环境变量启动校验已完成。
  - API Rate Limiting 已完成。
  - CI 已包含 Web 单测。
- Phase 7 / M17 主线：
  - `knowledge_search` 已从单一向量检索升级为 Hybrid Retrieval，默认 `balanced`。
  - `retrieveKnowledge` 已支持 `fast` / `balanced` / `deep`，`balanced` 走 semantic + keyword + metadata + RRF，`deep` 已包含 query rewrite / multi-query。
  - Memory MVP 已完成：`agentMemories` 数据模型、`memory_read`、确认式 `memory_write`、Memory Review UI、基础冲突处理和 Agent 自动注入。
  - Web Agent 写文档已完成安全闭环：`document_write` / `document_update` 默认 dry-run，前端确认后落库。
  - `web_extract` 已完成 URL 校验、私网拦截、超时、HTML 清洗和长度裁剪。
  - `document_search` 已完成文档元数据搜索。
  - 只读 tool 卡片已调整到 AI 回答上方，写类确认 tool 保持在回答下方。
  - ReAct 达到工具轮次上限后已强制进入最终回答阶段，避免无文本输出。

## 当前结论

- M17 主线已经基本完成：Agent tools、Memory MVP、Hybrid RAG、文档写入确认流、基础观测性和关键 UI 体验均已闭环。
- 当前真正的剩余重点已经从“Agent 主链路能不能跑通”切换为“质量、可观测性、评测和体验补齐”。
- 下一阶段不建议继续扩 Web Agent 写类能力，优先补跨请求缓存、Memory/RAG 质量、Trace sink 和测试/eval，否则能力面继续扩大后回归成本会快速上升。

1. 先做跨请求 Tool 结果缓存：将现有单次 run cache 扩展为 5 分钟 LRU，注意 user/workspace/query/tool arguments 维度隔离。
2. 补 Memory 增强：显式 Qdrant 同步、embedding 状态字段和 Memory eval，避免长期记忆与向量索引漂移。

4. 补 Agent Trace sink：把当前 `AgentTracer` 事件接入 Sentry span 或持久化事件表，为 M18 Harness replay 准备数据。
5. 补前端 AI 组件测试和 AI Chat E2E：优先覆盖 tool 展示位置、确认写入、重复 tool 折叠和滚动行为。
