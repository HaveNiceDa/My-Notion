# Web 设置页 PAT Scope 与过期时间配置

## 目标

在 PAT 创建表单中补齐权限范围和过期时间配置，让用户在 Web 设置页创建 token 时可以控制 `scopes` 与 `expiresAt`。

## 已完成

- 扩展 `SettingsModal` 的 PAT 创建表单：
  - 增加 `docs:read` 与 `docs:write` scope 复选项。
  - 默认选中 `docs:read` 与 `docs:write`，保持与后端默认行为一致。
  - 提交前校验至少选择一个 scope。
  - 增加 `datetime-local` 过期时间输入。
  - 过期时间留空时创建不过期 token。
  - 提交前校验过期时间必须晚于当前时间。
- 创建请求现在会向 `POST /api/cli/tokens` 传入：
  - `name`
  - `scopes`
  - `expiresAt`
- 保持现有一次性明文 token 流程：
  - 创建成功后仍只展示一次明文 token。
  - 未确认保存前仍阻止关闭设置弹窗。
- 补充中英文 i18n：
  - `packages/business/i18n/en.json`
  - `packages/business/i18n/zh-CN.json`

## 验证结果

- `pnpm --filter @notion/web exec tsc --noEmit`：通过。
- `pnpm --filter @notion/web lint`：通过，仍有 7 个历史 warning，非本次新增。
- `pnpm --filter @notion/web build`：通过，Sentry auth token/source map 为现有 warning。

## 已知风险

- scope 候选项目前只包含 `docs:read` 与 `docs:write`，后续如新增 `kb:*` 或 `agent:*` 需要扩展 UI 常量和文案。
- `datetime-local` 使用浏览器本地时区，服务端存储为毫秒时间戳。
- 尚未补 UI 自动化测试覆盖创建不同 scope/过期时间的流程。

## 下一步建议

- 可补 Playwright 用例覆盖 PAT 创建、scope 配置、过期时间校验、复制确认和撤销流程。
- 若当前 Web PAT 管理闭环足够，可进入 MCP Phase 4。
