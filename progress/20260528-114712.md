# Convex 文档写入函数注册修复

## 本轮目标

- 修复前端确认 `document_update` 时提示 `Could not find public function for 'documents:updateFromMarkdown'` 的问题。

## 根因

- Web Convex 入口 `apps/web/convex/documents.ts` 只使用 `export * from "@notion/convex/documents"`。
- 新增的 `createFromMarkdown` / `updateFromMarkdown` 位于共享包 `packages/convex` 内，运行中的 Convex dev 部署没有感知到新增 public mutation，导致前端可调用但后端函数注册表缺失。

## 已完成变更

- 在 `apps/web/convex/documents.ts` 中显式 re-export：
  - `createFromMarkdown`
  - `updateFromMarkdown`
- 重新执行 Convex codegen 并上传函数。

## 验证

- `pnpm --filter @notion/web exec convex codegen`：通过。
- `pnpm --filter @notion/web exec convex run documents:updateFromMarkdown '{"documentId":"nonexistent-id","contentMarkdown":"x"}'`：
  - 返回 `Unauthenticated`，说明 public function 已注册。
  - 不再返回 `Could not find public function`。
- `pnpm --filter @notion/web exec tsc --noEmit`：通过。
- `GetDiagnostics`：无诊断问题。

## 下一步建议

1. 如果浏览器仍复现旧错误，重启正在运行的 `convex dev` / Web dev server，清掉 dev server 旧函数注册缓存。
2. 后续新增共享包 Convex public function 时，优先在 `apps/web/convex/*.ts` 中显式 re-export，避免 watcher 漏感知。
