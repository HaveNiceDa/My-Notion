## 已知风险

- 自动提取器是 M27 保守规则版，只识别明确用户信号；更复杂的 LLM 提取、频控和真实 lifecycle 触发仍需后续增强。
- Memory Eval fixture 是小规模合成样本，能防主链路回归，但不能替代真实脱敏数据评估。
- Trace 仍是轻量 sink 机制，尚未持久化到 Convex 或提供 Trace Replay UI。
- Settings 当前展示策略说明和安全开关语义，尚未实现用户级持久化设置。

## 下一步建议

1. 引入真实脱敏 Memory/RAG eval 数据，持续扩充 `eval:memory`。
2. 将 Trace sink 持久化，建设 Tool Trace Replay。
3. 为自动提取增加频控、LLM 审核和用户级设置持久化。
4. 将 Memory policy 与后续 Web Agent MCP adapter 联动。
