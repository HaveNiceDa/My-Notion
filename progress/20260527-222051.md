# Memory Write 确认 UI

## 本轮目标

- 当 `memory_write` tool 返回 `confirmationRequired` 时，在 AI Chat tool 卡片中展示“保存记忆 / 取消”确认按钮。
- 用户点击确认后，由前端直接调用 Convex mutation 保存记忆。

## 已完成变更

- 更新 `apps/web/src/components/ai-chat/ToolCallCard.tsx`：
  - `MemoryWriteResult` 接入 `useMutation(api.agentMemories.createAgentMemory)`。
  - dry-run 且 `confirmationRequired=true` 时展示“保存记忆 / 取消”按钮。
  - 点击“保存记忆”后调用 Convex 写入，并展示保存中、已保存或错误状态。
  - 点击“取消”后仅本地标记为已取消，不写入数据库。
  - 保持非 dry-run 的已保存结果展示。
- 更新中英文 i18n：
  - `memorySaving`
  - `saveMemory`
  - `cancelMemory`
  - `memoryCancelled`
  - `memorySaveFailed`

## 已知缺口

- 保存/取消状态当前是 `ToolCallCard` 本地状态，刷新历史消息后不会保留按钮操作状态。
- 还没有 Memory Review UI，用户保存后不能在专门页面集中管理。
- 未新增前端组件单测；当前通过 TypeScript、lint、build 和 Agent tool 单测覆盖基础风险。

## 下一步建议

1. 做 Memory Review UI：展示、停用、编辑长期记忆。
2. 将已保存/已取消状态持久化到 assistant message 的 tool result，避免刷新后重复显示确认按钮。
3. 后续可把 memory read 自动注入 Agent loop，减少 LLM 忘记调用 `memory_read` 的概率。
