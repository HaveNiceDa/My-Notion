# M22.1/M22.2 Whiteboard Metadata And Read Path

## 目标

- 推进 M22 画板存储迁移的前两步：先建立 Convex 元数据模型和轻量读取契约，再改造 Web 端缩略图/全屏读取链路。
- 继续保持画板创建入口关闭，不在 R2 保存链路完成前恢复 Convex DB 大对象写入热路径。

## 已完成

- 扩展 `whiteboards` schema，新增 scene/thumbnail 对象存储元数据字段：object key/url/hash/bytes、`assetVersion`、`assetMigratedAt`。
- 保留 legacy `sceneJson` 与 `thumbnailDataUrl` 字段，保证旧数据和旧客户端仍可 fallback。
- 新增 `getPreviewById` 与 `listPreviewByDocument`，只返回标题、缩略图 URL、更新时间和 asset version，不返回 `sceneJson`、`thumbnailDataUrl` 或 `sourceDsl`。
- 新增 `getSceneById`，作为全屏编辑/显式读取完整 scene 的入口，优先暴露未来对象存储 URL，同时保留 legacy `sceneJson` fallback。
- Web 缩略图组件改为只订阅轻量 preview；未登录/公开预览场景不触发私有画板 query，继续使用 block props fallback。
- 全屏画板组件改为调用 `getSceneById`，并预留 `sceneObjectUrl` 客户端 fetch 逻辑；当前未重新接回缩略图点击入口，避免提前恢复 legacy 大对象写入。
- 插入画板命令的类型兼容 `thumbnailUrl`，后续 R2 缩略图 URL 可直接写入 block props。

## 验证

- `GetDiagnostics`：未发现新增诊断问题。
- `pnpm --filter @notion/web typecheck`：通过。
- `pnpm --filter @notion/web lint`：通过，仍有 7 个既有 warning，非本次新增。
- `pnpm --filter @notion/web build`：通过；Sentry auth token 缺失提示为本地构建环境既有提示。

## 已知限制

- 本阶段尚未新增 R2/S3 upload API，也未改造保存链路；`updateScene` 仍是 legacy DB 大对象写入接口。
- `getById/listByDocument` 保留完整返回以兼容旧调用方，后续 M22.3/M22.4 需要逐步替换并限制默认使用。
- CLI/MCP list/fetch/export 契约尚未改造，仍属于 M22.4 范围。
- 旧画板数据尚未迁移，`thumbnailUrl` 为空时缩略图会显示空画板占位或使用 block props fallback。

## 下一步

- M22.3：新增 R2/S3 封装与 `/api/whiteboard-assets/upload`，保存时先上传 scene/thumbnail，再调用 Convex metadata mutation。
- M22.3：为 `WhiteboardEditor` 增加 scene hash 去重，并把 autosave 从 1200ms 调整到 5000ms。
- M22.4：改造 CLI/MCP whiteboard list 为 summary，fetch/export 才读取完整 scene。
- M22.5：新增 dry-run 迁移脚本，上传 legacy scene/thumbnail 到 R2 并写回元数据。
