# Document Read Tool 接入

## 完成的改动

- 新增 `use-current-document-store`，用于在 Web 客户端保存当前文档上下文。
- 文档详情页会在加载、编辑内容变更、标题变更时同步当前 `documentId`、标题和内容到 store。
- AI Chat 发送 Agent 请求时会携带当前文档上下文。
- `/api/agent/stream` 新增 `document_read` tool 执行逻辑。
- Agent `auto` 模式遇到“当前页面 / 此页面 / 当前文档 / 总结 / 翻译 / 深度剖析”等意图时优先执行 `document_read`。
- `document_read` 使用 `@notion/ai/utils` 的 `extractTextFromDocument` 将 BlockNote JSON 内容转为纯文本。
- 消息列表补充 `document_read` tool 的展示文案。
- 当前文档 tool 仍沿用后端直接执行策略，不向 DashScope 发送 object/required `tool_choice`，避免 thinking mode 兼容问题。

## 已知缺口或风险

- `document_read` 当前通过客户端传入当前文档内容，适合当前侧边栏场景；后续如开放服务端独立调用，应改为基于 Convex auth 在服务端读取文档。
- Agent `auto` 仍是启发式规则，尚不是完整 LLM tool planner。
- 文档标题变更时 store 里的内容来自当前 Convex document 快照，若编辑器内容尚未完成防抖保存，标题事件不会携带最新编辑内容；内容编辑事件会同步最新内容。

## 下一步建议

- 将 `auto` 策略升级为可配置 tool planner。
