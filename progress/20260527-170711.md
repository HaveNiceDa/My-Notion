# Agent Memory MVP

## 本轮目标

- 按 M17 下一步计划实现 Memory MVP。
- 先打通结构化长期记忆的读写闭环，不接 Qdrant embedding，不做 Memory Review UI。

## 已完成变更

- 新增 `agentMemories` Convex schema：
  - 支持 `preference` / `project` / `episodic` 三类记忆。
  - 支持 `active` / `superseded` / `deleted` 状态。
  - 记录 `source`、`reason`、`confidence`、`expiresAt`、`embeddingRef`、`createdAt`、`updatedAt`。
  - 增加用户、类型和更新时间相关索引。
- 新增 Convex 函数：
  - `listAgentMemories`：按当前用户读取 active 记忆，支持 query/type/limit。
  - `createAgentMemory`：写入记忆，支持 supersede 旧记忆。
  - `deactivateAgentMemory`：软删除记忆。
- 新增 Web Convex 导出：
  - `apps/web/convex/agentMemories.ts`。
- Agent API 接入带 Clerk token 的 `ConvexHttpClient`：
  - 在 `/api/agent/stream` 中注入到 `ToolContext`。
- 新增 Agent tools：
  - `memory_read`：读取长期记忆。
  - `memory_write`：默认 dry-run，返回 `confirmationRequired`，只有 `dryRun=false` 才真实写入。
- 更新 tool registry：
  - `memory_read` / `memory_write` 始终可用。
- 更新 AI Chat tool 可视化：
  - 展示读取到的记忆列表。
  - 展示记忆写入预览、确认要求和保存状态。
- 更新中英文 i18n 文案。
- 补充 Agent tool 单测：
  - 覆盖 memory read 缺 Convex client、正常读取、write dry-run、write 落库、空内容错误。

## 已知缺口

- `memory_write` 的真实写入仍依赖 LLM 遵守 `dryRun=false` 只在用户确认后使用；还没有前端确认按钮闭环。
- Memory 仅结构化存储和轻量 token match，不接 Qdrant embedding。
- 还没有 Memory Review UI，用户无法在界面集中查看、编辑和删除记忆。
- `memory_read` 由 LLM 自主调用，尚未做每轮自动注入用户偏好。

## 下一步建议

1. 做写入确认 UI：当 `memory_write` 返回 `confirmationRequired` 时，前端提供“保存记忆/取消”操作。
2. 做 Memory Review UI：允许用户查看、停用和编辑长期记忆。
3. 将 Memory 接入检索：为 active memory 建 embedding，或先在 Agent loop 开始前自动注入最近/相关记忆。
