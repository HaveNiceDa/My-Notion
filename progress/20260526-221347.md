# Phase 5 P1：Skills 同步漂移校验

## 目标

为 `packages/my-notion-skills` 到 `.trae/skills` 的同步流程增加只读校验模式，防止源 skill 与已安装 skill 内容发生漂移。

## 已完成

- 扩展 `scripts/sync-my-notion-skills.mjs`：
  - 保留默认同步模式。
  - 新增 `--check` 只读校验模式。
  - 比较源目录与目标目录的文件集合和文件内容。
  - 忽略 `.gitkeep`。
  - 检测缺失文件、额外文件、内容不一致文件。
  - 发现漂移时输出 JSON 差异并返回非 0。
- 更新根 `package.json`：
  - 新增 `pnpm sync:skills:check`。
- 更新 `docs/my-notion-cli-release-checklist.md`：
  - 发布前命令加入 `pnpm sync:skills:check`。
  - Skills 同步检查加入漂移校验要求。
- 更新架构文档和 `.trae` plan：
  - Phase 5 P1 的 skills 漂移校验项标记完成。

## 验证结果

- `node --check scripts/sync-my-notion-skills.mjs`：通过。
- `pnpm sync:skills`：通过。
- `pnpm sync:skills:check`：通过，所有 skill `diffCount` 为 0。
- 负向漂移验证：临时修改 `.trae/skills/my-notion-docs/SKILL.md` 后，`pnpm sync:skills:check` 返回非 0，并报告 `modified my-notion-docs/SKILL.md`。
- 恢复验证：重新运行 `pnpm sync:skills && pnpm sync:skills:check` 后通过，确认测试污染已清理。
