# 强类型 Sources 与流式续跑协议设计

## 目标

继续推进 M21/M22 之间的治理工作：将 Tool result `sources` 收敛为强类型 discriminated union，并设计完整流式续跑 checkpoint/resume 协议。

## 已完成

- `ToolResultSource` 强类型化：
  - 新增 `DocumentToolResultSource`
  - 新增 `WebToolResultSource`
  - 新增 `MemoryToolResultSource`
  - `ToolResultSource` 使用 `type` 作为 discriminant。
- `withToolResultContract` 和 `buildToolErrorResult` 的 `sources` 参数从 `unknown[]` 收紧为 `ToolResultSource[]`。
- 迁移并验证现有来源：
  - `document_search` / `document_read` / `knowledge_search` 使用 `type: "document"`。
  - `web_search` / `web_extract` 使用 `type: "web"`。
  - `memory_search` / `memory_write` 使用 `type: "memory"`。
  - `mcp_my_notion_call` 文档工具使用 `type: "document"`。
- 前端 `types.ts` 增加 `ToolResultEnvelope`，为后续 UI 展示 typed sources 提供类型入口。
- 流式续跑协议：
  - 新增 `docs/agent-stream-resume-protocol.md`。
  - 定义 `runId`、`seq`、`AgentStreamEnvelope`、`AgentStreamResumeCursor`、checkpoint kind、resume 请求和 replay 规则。
  - 明确只读工具、写入预览工具、已确认写入状态的恢复边界。
- 代码类型入口：
  - `apps/web/src/lib/agent/stream.ts` 新增 `AgentCheckpointKind`、`AgentStreamEnvelope`、`AgentStreamResumeCursor`、`AgentResumeControlEvent`。
  - `apps/web/src/components/ai-chat/types.ts` 新增对应前端 cursor/checkpoint 类型。
- 文档索引：
  - `docs/README.md` 增加流式续跑协议入口。

## 已知限制

- 当前完成的是强类型契约和协议设计，尚未实现 `agentRuns` / `agentRunEvents` / `agentRunCheckpoints` 持久化表。
- 前端还未展示 typed sources，只是已有类型入口。
