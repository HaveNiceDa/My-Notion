# PAT 创建流程 Playwright 测试去 Clerk 依赖

## 目标

简化 PAT 创建流程 Playwright 测试，去掉对 Clerk 登录态、`auth-setup` 和 `.auth/storage-state.json` 的依赖，使用 mock 数据直接跑通设置页 PAT 创建流程。

## 已完成

- 新增测试专用页面：
  - `apps/web/src/app/[locale]/(marketing)/(routes)/test-pat-settings/page.tsx`
  - 页面只负责通过 `useSettings` 打开已有 `SettingsModal`。
  - 该页面位于 marketing route 下，不经过 `(main)` layout 的 Clerk/Convex auth redirect。
- 更新 `tests/web/pat-settings.spec.ts`：
  - 访问 `/en/test-pat-settings`，不再访问受保护的 `/en/documents`。
  - 继续使用 `page.route("**/api/cli/tokens")` mock `GET` 与 `POST`。
  - 保留 scope 选择校验、过期时间校验、创建请求体断言和一次性明文 token 流程。
  - 使用 `context.grantPermissions(["clipboard-write"])` 支持复制按钮路径。
- 更新 `playwright.config.ts`：
  - `pat-settings.spec.ts` 回到普通 `chromium` project。
  - 不再挂到 `chromium-authenticated`，因此不会触发 `auth-setup`。

## 验证结果

- `pnpm exec playwright test tests/web/pat-settings.spec.ts --list`：通过，确认只挂在 `chromium` project。
- `pnpm exec playwright test tests/web/pat-settings.spec.ts --project chromium`：通过。
- `pnpm --filter @notion/web exec tsc --noEmit`：通过。
- `pnpm --filter @notion/web lint`：通过，仍有 7 个历史 warning，非本次新增。

## 处理过的问题

- 最初使用 `__tests__` 目录作为路由段会被 Next.js 视为私有目录，导致 404；已改为 `test-pat-settings`。
- 测试页最初把整个 Zustand settings store 作为 `useEffect` 依赖，触发最大更新深度错误；已改为只订阅稳定的 `onOpen` selector。

## 下一步建议

- 后续可把更多纯 UI 校验迁移到这种测试专用页面，减少对 Clerk 测试凭据的依赖。
- 若要验证真实 token API 与 Clerk 登录态，则保留 `chromium-authenticated` project 作为集成测试层。
