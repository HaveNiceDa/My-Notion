# API Token 与 Convex HTTP Actions 最小闭环

## 目标

为 My-Notion CLI / Agent 生态建立首个机器访问闭环：登录态用户可以创建 Personal Access Token，外部调用方可以用 Bearer token 访问 Convex HTTP Actions 完成文档创建、读取、搜索和更新。

## 已完成

- 新增 `apiTokens` Convex schema，仅保存 token hash、prefix、scope、过期/撤销/最近使用时间等元信息。
- 新增 `@notion/convex/cli` 模块，包含 token 记录、token 认证、使用时间更新和 CLI 文档内部操作。
- 新增 `apps/web/convex/http.ts`，挂载 `/cli/v1/auth/status` 与 `/cli/v1/documents` 系列 HTTP Actions。
- 新增 `POST /api/cli/tokens`，在 Clerk 登录态下生成 `mnt_` 前缀 PAT，明文 token 只在创建响应中返回一次。
- 刷新 Convex 生成类型，使 `api.cli` 与 `internal.cli` 可用。

## 已知风险

- Markdown 到 BlockNote 的转换目前是最小 paragraph 兼容实现，不保留标题、列表、代码块等结构化语义。
- `POST /api/cli/tokens` 目前只实现创建 token，尚未实现 token 列表、撤销、过期管理 UI。
- Convex HTTP Actions 使用 Convex `.site` 域名访问；生产部署时需要确认前端向用户展示的 API URL。
- HTTP Actions 尚未加入专用限流和审计日志，后续应与现有 Agent rate limiter 策略对齐。
