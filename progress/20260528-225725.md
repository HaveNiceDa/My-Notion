# Retrieval Eval Golden Set

## 本轮目标

- 构造固定问题集，验证当前 `hybrid retrieval + context packing + citation quality` 的效果。
- 用 synthetic corpus 做可重复测试，不依赖真实 Qdrant、embedding 或线上文档数据。
- 为后续 M18 Harness / Eval 预留最小 golden set 形态。

## 已完成变更

- 新增 `packages/ai/__tests__/retrieval-eval.test.ts`。
- 构造 3 个固定问题：
  - `hybrid-api-contract`：验证 semantic + keyword 多路召回融合，并确认相邻 API chunk 被 packing 合并。
  - `metadata-recency`：验证 metadata recall 对 Memory / Qdrant 同步类问题的补召回作用。
  - `token-budget-truncation`：验证长上下文在较小 `contextTokenBudget` 下被截断，并触发 `needsMoreRetrieval`。
- 每个 eval case 使用固定 synthetic corpus，分别指定 `semanticScore`、`keywordScore`、`metadataScore`。
- Eval 断言指标：
  - 是否命中期望 `documentId`。
  - `citationCoverage` 是否达到阈值。
  - `uniqueDocumentCount` 是否达到阈值。
  - `packedCount` 是否符合预期。
  - `contextTruncated` 是否符合预期。
  - `packing.mergedItemCount` 是否达到预期。
  - `sourceCoverage` 是否覆盖期望召回源。
  - `needsMoreRetrieval` 是否符合质量判断。
- 更新 `docs/ai-chat-refactor-plan.md`：
  - RAG 当前状态补充 synthetic golden set。
  - Harness 状态更新为已有最小 retrieval eval golden set。
  - 后续重点调整为真实数据 retrieval eval、memory eval、tool trace replay 和 CI smoke。

## Eval 结果

- `hybrid-api-contract`：通过。
  - 证明 keyword + semantic 可融合到同一 API 文档 chunk。
  - 相邻 chunk 可被 Context Packing 合并。
  - Citation Quality 判断无需补充检索。
- `metadata-recency`：通过。
  - 证明 metadata recall 能补强“最近更新 / 同步状态”类问题。
  - Memory Qdrant 同步文档相邻 chunk 可被合并。
- `token-budget-truncation`：通过。
  - 证明 token budget 会触发上下文截断。
  - Citation Quality 能标记 `needsMoreRetrieval=true`。

## 已知风险

- 当前 golden set 是 synthetic corpus，能验证算法链路和质量信号，但不能代表真实用户数据分布。
- 还没有计算严格的 `recall@k`、`MRR`、`nDCG` 等 IR 指标。
- 尚未接入 CI smoke job，也未形成独立 `pnpm eval:retrieval` 命令。
- `deep` 策略和 Query Rewrite 的 eval 仍较少，目前主要覆盖 `balanced` 主路径。

## 下一步建议

1. 将 eval case 抽成 JSON/TS fixture，便于扩展真实样本和失败样本。
2. 增加 `recall@k`、`MRR`、citation coverage、truncation rate 等聚合指标。
3. 接入 CI smoke 或新增 `pnpm eval:retrieval`。
4. 根据真实数据 eval 结果再决定是否引入二阶段 rerank。
