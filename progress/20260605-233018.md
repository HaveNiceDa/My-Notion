# M20 Web Agent MCP Adapter 最小闭环

## 目标

落地 M20 第一版：让 Web Agent 通过受控 adapter 调用 My-Notion MCP 工具能力，同时继续遵守写入预览与用户确认链路。

## 已完成

- 新增 `mcp_my_notion_call` Agent Tool，接入 Web Agent 现有 registry 与 ReAct Loop。
- 新增 My-Notion MCP adapter 白名单，仅开放：
  - `my_notion_docs_search`
  - `my_notion_docs_fetch`
  - `my_notion_docs_create`
  - `my_notion_docs_update`
- 写入类 MCP 工具在 adapter 层强制 `dryRun: true`，即使模型传入 `dryRun: false` 也只返回预览。
- MCP create/update 复用现有 `document_write` / `document_update` 预览结果结构，让前端继续通过现有确认按钮触发真实 Convex mutation。
- MCP fetch 使用 Web runtime 内部轻量 Markdown 转换，避免从 Next API route 导入 Convex server 侧模块。
- `ToolCallCard` 增加 MCP adapter 展示逻辑；读工具展示摘要、文档链接和 Markdown，写工具复用文档写入确认 UI。
- 补充中英文 i18n key 与单元测试，覆盖白名单、search、create dry-run、update dry-run。

- `pnpm --filter @notion/web build`：通过；Sentry auth token 缺失提示为本地构建环境 warning。
- VS Code diagnostics：新增/修改核心文件无错误。

## 已知限制

- 第一版是 `in_process` 受控 adapter，不连接任意外部 MCP server，也不启动 STDIO 子进程。
- 当前只开放文档 MCP 工具；画板 MCP 工具因 Convex 带宽止血策略暂不接入 Web Agent。
- `docs_fetch` 的 Markdown 转换为轻量实现，复杂 BlockNote 结构后续可按真实 case 增强。
- Tool 结果契约已带 adapter metadata，但更完整的 `summary/sources/metadata/recoverable` 统一仍属于 M21。
